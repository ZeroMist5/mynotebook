面试题相关

1. EventLoop

1.说明

  1. JavaScript是一门单线程执行的编程语言，也就是说，同一时间只能做一件事情。

  2. 单线程执行任务队列的问题

    如果前一个任务非常耗时，则后续的任务就不得不一直等待，从而导致程序假死的问题。

2. 同步任务和异步任务

为了防止某个耗时任务导致程序假死的问题，JavaScript 把待执行的任务分为了两类：

① 同步任务（synchronous）

⚫ 又叫做非耗时任务，指的是在主线程上排队执行的那些任务

⚫ 只有前一个任务执行完毕，才能执行后一个任务

② 异步任务（asynchronous）

⚫ 又叫做耗时任务，异步任务由 JavaScript 委托给宿主环境进行执行.

⚫ 当异步任务执行完成后，会通知 JavaScript 主线程执行异步任务的回调函数.

3.两种任务的执行过程

① 同步任务由 JavaScript 主线程次序执行

② 异步任务委托给宿主环境执行

③ 已完成的异步任务对应的回调函数，会被加入到任务队列中等待执行

④ JavaScript 主线程的执行栈被清空后，会读取任务队列中的回调函数，次序执行

⑤ JavaScript 主线程不断重复上面的第 4 步

4.事件循环

  1. JavaScript 主线程从“任务队列”中读取异步任务的回调函数，放到执行栈中依次执行。
  2. 这个过程是循环不断的，所以整个的这种运行机制又称为 EventLoop（事件循环）。

2. 宏任务与微任务

2.1 什么是宏任务和微任务

JavaScript 把异步任务又做了进一步的划分，异步任务又分为两类，分别是：

① 宏任务（macrotask）

​ ⚫ 异步 Ajax 请求、

​ ⚫ setTimeout、setInterval、

​ ⚫ DOM事件回调

​ ⚫ 文件操作

​ ⚫ 其它宏任务

② 微任务（microtask） 

 ⚫ Promise.then、.catch 和 .finally

​ ⚫ process.nextTick

​ ⚫ MutationObserver的回调

​ ⚫ 其它微任务

2.2 宏任务和微任务的执行顺序

  1. 每一个宏任务执行完之后，都会检查是否存在待执行的微任务，

  2. 如果有，则执行完所有微任务之后，再继续执行下一个宏任务。

2.3 举例说明两种任务场景

① 小芳和小明去银行办业务。首先，需要取号之后进行排队

​ ⚫ 宏任务队列

② 假设当前银行网点只有一个柜员，小芳在办理存款业务时，小明只能等待

​ ⚫ 单线程，宏任务按次序执行

③ 小芳办完存款业务后，柜员询问她是否还想办理其它业务？

​ ⚫ 当前宏任务执行完，检查是否有微任务

④ 小芳告诉柜员：想要买理财产品、再办个信用卡、最后再兑换点猴年纪念币？

​ ⚫ 执行微任务，后续宏任务被推迟

⑤ 小芳离开柜台后，柜员开始为小明办理业务

​ ⚫ 所有微任务执行完毕，开始执行下一个宏任务

3. 两种风格的接口

3.1 非restful(restless)API

特点：

  1. 请求方式不决定请求的CRUD操作
  2. 一个请求路径只对应一个操作
  3. 一般只有GET/POST
请求方式 目标址 操作
GET http://127.0.0.1:3000 查看主页面
GET http://127.0.0.1:3000/list 查看列表页
GET http://127.0.0.1:3000/delOne?id=10 删除某一项数据
GET http://127.0.0.1:3000/checkOne?id=12 查询一条数据
POST http://127.0.0.1:3000/updateOne 更新数据
POST http://127.0.0.1:3000/addOne 增加数据

3.2 REST(restful) API

请求方式 目标地址 操作
GET http://127.0.0.1:3000 查询主页面
GET http://127.0.0.1:3000/list/1 查询列表页一条数据
DELETE http://127.0.0.1:3000/list/1 删除一条数据
PUT http://127.0.0.1:3000/list/1 更新这条数据的全部内容
PETCH http://127.0.0.1:3000/list/1 更新这条数据的局部内容

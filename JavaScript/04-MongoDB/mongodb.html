MongoDB

简介

MongoDB 是为快速开发互联网Web应用而设计的数据库系统，官方地址 https://www.mongodb.com/

数据库（DataBase）是按照数据结构来组织、存储和管理数据的仓库。是一个应用程序

学习数据库的原因：

  • 易于管理大量的数据
  • 方便后期的更新维护
  • 更加的安全

下载安装

下载地址 https://www.mongodb.com/download-center/community

安装过程截图

安装的默认位置

C:\Program Files\MongoDB

安装完毕后进行几步操作

一、为了方便在命令行下运行，可以配置 mongodb 命令的环境变量 PATH

此电脑 -> 属性 -> 高级系统设置 -> 环境变量 -> 双击 Path -> 新建 -> 设置 mongod.exe 所在文件夹路径

C:\Program Files\MongoDB\Server\3.2\bin

二、创建默认的仓库文件夹

c:\data\db

三、打开命令行窗口输入mongod 启动数据库服务器

四、另起一个命令行运行 mongo

使用

三个重要概念

  • 一个服务可以创建多个数据库
  • 数据库（database） 数据库是一个仓库，在仓库中可以存放集合
  • 集合（collection） 集合类似于JS中的数组，在集合中可以存放文档
  • 文档（document） 文档数据库中的最小单位，类似于 JS 中的对象，在 MongoDB 中每一条数据都是一个 JS 的对象

常用命令

数据库集合命令

  1. 启用数据库

    先打开一个命令行终端 输入: mongod    表示启动数据库服务器
再打开另一个命令行终端 输入: mongo   表示进入mongo数据库程序

  2. 显示所有的数据库

    show databases   查看当前软件中的所有的数据库
show dbs   上面命令的简写

  3. （创建）切换到指定的数据库

    use 数据库名

  4. 显示当前所在的数据库

    db

  5. 删除当前数据库（先切换再删除）

    use project_1
db.dropDatabase()

  6. 显示当前数据库中的所有集合

    db.createCollection('users');     //创建集合
show collections

  7. 删除当前集合

    db.collection.drop()
比如： db.stu.drop()

  8. 重命名集合

    db.collection.renameCollection('newName')
比如： db.stu.renameCollectiion('stua')

操作集合时，如果集合不存在则会自动创建集合

文档命令

1）插入文档

db.collection.insert(文档对象);
比如: db.users.insert({name:'',age:20})
db.users.inserMany([{name:'rose',age:20},{name:'jack',address:'武汉市'},{name:'jerry',score:100}])   // 表示一次性插入多条数据

2) 查询文档

db.collection.find(查询条件)    
db.user.find()  // 查看所有的数据
db.user.find({age:20})  // 查找所有的符合条件的
db.collection.findOne(查询条件)
db.user.find({age:20}) // 只查找一条符合条件的

3) 更新文档

db.collection.update(查询条件,新的文档)   
db.user.update({name:'jack'},{name:'jackaa'})  // 会丢掉其他属性
// 更新一个
db.collection.updateOne(查询条件,要更新的内容) 
db.user.updateOne({name:'jack'},{$set:{age:27}})
// 批量更新
db.collection.updateMany(查询条件,要更新的内容)
db.user.updateMany({address:null},{$unset:{address:''}})  // 某些字段属性为null则可以一次性删除
//eg
db.students.update({name:'jack'},{$set:{age:19}})

4) 删除集合中的文档

db.collection.remove(查询条件)
比如: db.users.remove({name:'aaa'})

条件控制

运算符

在 mongodb 不能 > < >= <= !== 等运算符，需要使用替代符号

  • > 使用 $gt great than

  • < 使用 $lt less than

  • >= 使用 $gte e equal

  • <= 使用 $lte

  • !== 使用 $ne

     db.user.find({age:{$gte:20,$lte:22}})  // 查找年龄大于20  小于22的所有的数据
逻辑或

$in 满足其中某个的找出来，有可能找出来的是多个

db.user.find({age:{$in:[18,24,26]}}) //  /[aedf]/

$or 逻辑或的情况

db.user.find({$or:[{age:18},{age:24}]});

$and 逻辑与的情况

 db.user.find({age:{$gte:20,$lte:22}})
db.students.find({$and: [{age: {$lt:20}}, {age: {$gt: 15}}]});
正则匹配

条件中可以直接使用 JS 的正则语法

db.user.find({name:/imissyou/});

Mongoose

介绍

Mongoose 是一个对象文档模型（ODM）库，它对Node原生的MongoDB模块进行了进一步的优化封装，并提供了更多的功能。 官网 http://www.mongoosejs.net/

作用

使用代码操作 mongodb 数据库

使用流程

一、安装 mongoose

在命令行下使用 npm 或者其他包管理工具安装（cnpm yarn）

npm install mongoose@5 --save

二、引入包

在运行文件中引入 mongoose

var mongoose = require('mongoose');

三、连接数据库

mongoose.connect('mongodb://127.0.0.1:27017/data');

//如果启动时遇到警告提醒， 则按照提示增加选项即可
mongoose.connect('mongodb://127.0.0.1:27017/data', {useNewUrlParser: true, useUnifiedTopology: true});

四、监听连接事件

mongoose.connection.on('open', function () {

    //下面编写数据库操作代码

    //五、创建文档结构
    var SongSchema = new mongoose.Schema({
        title: String,  //歌名
        author: String  //歌手
    });

    //六、创建文档模型
    var SongModel = mongoose.model('songs', SongSchema);

    //七、使用模型进行文档处理（这里以增加数据为例）
    SongModel.create({title:'野狼disco',author:'宝石gem'}, function(err,data){
        if(err) throw err; //这里判断错误

        //下面编写创建成功后的逻辑
        // ... ...

        //八、关闭数据库连接（可选，代码上线之后一般不加）
        mongoose.connection.close();
    });

});

文档结构可选的字段类型列表

  • ==String==
  • ==Number==

关系型数据库（RDBS）

代表有：MySQL、Oracle、DB2、SQL Server...

特点：关系紧密，都是表

优点：

1、易于维护：都是使用表结构，格式一致；

2、使用方便：通用，可用于复杂查询；

3、高级查询：可用于一个表以及多个表之间非常复杂的查询。

缺点：

1、读写性能比较差，尤其是海量数据的高效率读写；

2、有固定的表结构，字段不可随意更改，灵活度稍欠；

3、高并发读写需求，传统关系型数据库来说，硬盘I/O是一个很大的瓶颈。

非关系型数据库（NoSQL not only SQL ）

代表有：MongoDB、Redis...

特点：关系不紧密，有文档，有键值对

优点：

1、格式灵活：存储数据的格式可以是key,value形式。

2、速度快：nosql可以内存作为载体，而关系型数据库只能使用硬盘；

3、易用：nosql数据库部署简单。

缺点：

1、不支持事务；

2、复杂查询时语句过于繁琐。

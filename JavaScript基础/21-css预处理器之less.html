21-CSS预处理器之less

什么是css预处理器

CSS 预处理器是一个特殊的编程语言，能够编写独有的语法来扩展CSS的语法。

作用：让 CSS 更容易维护,比写起传统的CSS更爽点。

最流行的 CSS 预处理器

  • Sass
  • LESS
  • Stylus

这里以讲解less为例。 less中文文档：https://less.bootcss.com/ 官网： https://lesscss.org/

Less 到底为 CSS 添加了什么功能？

  • 变量声明
  • 嵌套规则
  • 混合(mixin)
  • 父元素选择器&
  • 导入样式
  • ...

安装 Easy LESS 编译器

浏览器默认不识别.less的文件，所以需要借助编译器将less转为原生的css。

可以安装easy less插件进行转换

基本使用：

示例： 项目中创建index.less文件,内容如下：

@width: 10px;
@height: @width + 10px;

#box {
  width: @width;
  height: @height;
}

保存后自动会在当前目录生成index.css的文件。

// index.css
#box {   
  width: 10px;   
  height: 20px; 
}

导入样式：

// index.html
<head>
    <link rel="stylesheet" href="./index.css">
</head>

配置less编译选项

安装上面插件后，可以进行个性化的配置，less参考配置如下：

配置文件路径：File -> Preferences -> settings 
切换:settings.json文件模式
{
    "window.zoomLevel": 2,
    "files.autoSave": "afterDelay",
    "workbench.sideBar.location": "right",
    "terminal.integrated.shell.windows": "C:\\Windows\\System32\\cmd.exe",
    "less.compile": {            // less相关配置
        "compress": true, // 压缩。即是否删除多余空白字符
        "sourceMap": false, // 是否创建相应的map文件，.map 文件主要用来调试源代码
        "out": "./css/", // 输出的目录名
        "outExt": ".css", // 输出文件的后缀,默认为.css
    }
}

配置文件说明

上面的配置是固定在编辑器中的默认配置，即任何项目都是采用同样的配置。

但是有些项目中目录结构是不一样的，所以上面的写法就固定死了，不够灵活。

我们希望同一个项目在在不同电脑中的vscode编辑器中打开都是采用同样的配置。。

如何配置呢？

我们只需在项目根目录中创建.vscode文件夹,里面创建settings.json文件,这样配置可以跟着项目走。

后面项目在任何电脑打开时，vscode都是同一套配置了。

注意配置文件的优先级：

当前项目配置(.vscode/settings.json) > vscode用户默认配置（settings.json） > 插件默认配置

通过后面学习的打包工具如：gulp、webpack也可以将scss或less编译成浏览器可以识别的css。

less常用的功能

变量声明

通过变量声明的方式，减少了原来在 CSS 中无法避免的硬编码重复。硬编码相当于代码写死了不够灵活。

示例： 创建index.less文件,内容如下：

@width: 10px;
@height: @width + 10px;

#box {
  width: @width;
  height: @height;
}

编译为：

#box {   
  width: 10px;  
  height: 20px; 
}

变量的作用域：

Less 中的作用域与 CSS 中的作用域非常类似。首先在本地查找变量，如果找不到，则从“父”级作用域继承。

@var: red;

#box {
  @var: green;
  #list {
    color: @var; // green
  }
}

嵌套

Less 提供了使用嵌套代替传统的层叠能力。 作用： 避免了重复输入父选择器，让复杂的 CSS 结构层次更清晰也便于维护。

假设我们有以下 CSS 代码：

#box {   
  color: red; 
} 
#box .header {   
  font-size: 12px; 
} 
#box .footer ul li a {   
  width: 300px; 
}

用 Less 语言我们可以这样书写代码：

#box {   
  color: red;   
  .header {     
    font-size: 12px;   
  }   
  .footer {     
    width: 300px;   
      ul {
          li {
              a {
                  width: 300px; 
              }
          }
      }
  } 
}

引用父选择器

& 表示当前选择器的父级

a {
  color: pink;
  &:hover {
    color: skyblue;
  }
}

最终编译为：

a {
  color: pink;
}
a:hover {
  color: skyblue;
}

混合（Mixins）

混合（Mixin）是一种将一组属性从一个规则集包含（或混入）到另一个规则集的方法。

假设我们定义了一个类（class）如下：

.bordered {   
  border-top: dotted 1px black;   
  border-bottom: solid 2px black; 
}

如果我们希望在其它规则集中使用这些属性呢？ 没问题，我们只需像下面这样输入所需属性的类名称即可，如下所示：

#menu a {   
  color: #111;   
  .bordered(); 
} 

.post a {   
  color: red;   
  .bordered(); 
}

.bordered 类所包含的属性就将同时出现在 #menu a 和 .post a 中了。

导入样式

若某一个样式文件被其他地方同时使用，可以@import语法进行导入。

你可以导入一个 .less 文件，此文件中的所有变量就可以全部使用了。如果导入的文件是 .less 扩展名，则可以将扩展名省略掉

// common.less
@bgColor: yellow;
.flex-c-c {
  display: flex;
  justify-content: center;
  align-items: center;
}
.line_ellipsis_1 {
  /*1.先强制一行内显示文本*/
  white-space: nowrap;
  /*2.超出的部分隐藏*/
  overflow: hidden;
  /*3.文字用省略号替代超出的部分*/
  text-overflow: ellipsis;
}
.line_ellipsis_2 {
  overflow: hidden;
  text-overflow: ellipsis;
  /*弹性伸缩盒子模型显示*/
  display: -webkit-box;
  /*限制在一个块元素显示的文本的行数*/
  -webkit-line-clamp: 2;
  /*设置或检索伸缩盒对象的子元素的排列方式*/
  -webkit-box-orient: vertical;
}

导入上面的文件：

// index.less
@import "common"; // 或 common.less

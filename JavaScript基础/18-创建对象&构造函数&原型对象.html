18-创建对象&构造函数&原型对象

复习：

DOM、闭包、call、apply、bind

闭包：

  • what: 指有权访问另一个函数作用域中的变量的函数。
  • why: 作用保存变量。具体应用场景：IIFE(闭包), 一次性函数，缓存函数
  • how: 注意内存泄漏问题。如何产生闭包？子函数访问祖先函数中的标识符（变量名或函数名）就会立即产生闭包函数

call/apply/bind:

  • what/why:改变函数内this的指向
  • 区别：
    • call和apply是立即执行，bind而是返回函数
    • call和bind是一个一个传递，而apply需要传递一个参数数组
  • how: 注意：第一个如果不需要传递伪造的对象，一般建议传递null，作为对象的占位符

arguments:

​ 一句话：获取函数所有的实参集合（伪数组）。

​ 将伪数组转化为真数组： Array.from(伪数组)

this指向：

​ 永远指向最后的调用者！不要看定义的位置，要看执行的环境。

DOM和事件：

  • DOM0和DOM2
  • 事件流：捕获->目标->冒泡
  • 阻止冒泡和默认行为
    • 阻止冒泡： event.stopPropagation(), event.cancelBubble = true
    • 阻止默认行为：event.preventDefault(), event.returnValue = false
  • 找节点
    • document.body => body

    • document.getElementById()
    • document.getElementsByClassName()
    • document.getElementsByTagName()

  • offsetLeft: 获取距离具有定位的祖先元素的上偏移

什么是对象

现实生活中：万物皆对象，对象是一个具体的事物，一个具体的事物就会有特征和行为。

如：一辆车，一个人，一只狗他们都是对象！！

车的属性： 颜色、重量。 行为有：启动、刹车、前进、后退。

人的属性： 身高、年龄、体重、姓名。 行为有： 说话、跑步、跳跃、打架、打游戏。

狗的属性：颜色、名字。 行为有：摇尾巴、汪汪叫等

在编程的世界中，特征叫做属性，行为叫做方法

为什么需要对象

如果使用基本数据类型的数据，我们所创建的变量都是独立的，值和值之间没有任何的关联，不能成为一个整体。

而对象是属于一种复合的数据类型，在对象中可以保存多个不同类型的数据。

这就是对象存在的意义。

如，用JS来表示一个人的信息

var name = "大锤";
var sex = "男";
var age = 28;

上面的变量都是独立的，没有任何关联，不能成为一个整体。

JS对象分类

  1. 内建对象
    • 由ES标准中定义的对象，任何JS引擎实现此标准的都可以使用
    • 比如： String、Number、Math、Boolean、Function、Object。
  2. 宿主对象
    • 有JS所在运行环境提供的对象，如浏览器环境
    • 比如： DOM、BOM对象
  3. 自定义对象
    • 开发人员自己创建的对象

      创建对象的多种方式

通过Object构造函数创建对象

有两种形式：

  • 字面量形式
  • 构造函数形式

示例：

创建一个对象，有属性name和方法hobby

// 字面量形式 【推荐，开发中使用最多】
var obj1 = {
  name: '科比',
  age: 18,
  hobby: function(){
    console.log('篮球贼厉害');
  }
}


// 构造函数形式
var obj = new Object();
obj.name = '科比';
obj.age = 18;
obj.hobby = function(){
    console.log('篮球贼厉害');
}

属性和方法的访问：

// 访问： 对象名.属性名
console.log( obj.name )

//访问函数：  对象名.函数名()
obj.hobby()

上面创建对象的缺点：

每次创建对象都需要给对象设置很多属性，如果创建的对象比较多，就会设置相同的属性名，代码冗余，而且后期难维护。

我们的期望是每创建一个对象不需要重复设置属性名，只需要传入属性值自动帮助我们设置。

解决办法：通过工厂方式创建。

通过工厂方式创建对象

工厂顾名思义就是加工产品的地方，在这里我们需要工厂帮助我们创建一个对象。

通过工厂方式创建对象核心思想：创建一个函数，调用函数时传递对象所需要的属性值，最终把对象返回即可。

// 创建一个工厂（函数），返回对象
function createObj(name,age){
  var obj = new Object();
  obj.name = name;
  obj.age = age;
  return obj;
}

var obj1 = createObj('大锤',38)
var obj2 = createObj('小锤',18)

工厂方式创建对象的缺点

创建对象的类型比较单一，也就是只能创建Object类型的对象。

但是我想创建人类型的对象、车类型的对象，js就没有办法实现了。

即没办法通过instanceOf来检测对象的类型

幸运的是，在js中可以通过自定义构造函数（构造器）形式来创建指定类型的对象。

通过自定义构造函数创建对象

自定义构造函数（构造器）

分类：

  • 内置构造函数（String、Number, Boolean、Function、Array、Error、RegExp、Object）
  • 自定义构造函数：Person

内置的Math没有构造器（constructor）,所以不能new操作

构造函数：就是把对象一些相同的属性和方法抽象出来封装到函数中，再通过此函数来创建对象。此函数就称之为是构造函数。

如下创建一个Person构造函数

function Person(name, age){
  // this = {}; // 默认会隐式创建一个this变量,代表当前创建的对象
  this.name = name;
  this.age = age;
  this.hobby = function(){
    console.log('爱打篮球')
  }
 // new 构造函数时，会隐式返回this关键字，即 return this;
}

// 通过new 关键字来创建对象
var p1 = new Person('欧文', 22);
console.log(p1.name); // 欧文
console.log(p1.hobby); // 爱打篮球

// 判断是否是Person构造函数的实例对象
console.log(p1 instanceof Person); // true

注意：

  1. 构造函数首字母一般约定大写。
  2. 构造函数要和new一起使用才有意义。
  3. new 构造函数时，内部会隐式返回this关键字

new 到底干了什么

我们知道通过new关键字可以创建一个对象。

其在new 的时候，主要做了以下3件事情

  1. 执行构造函数中的代码
  2. 构造函数会在堆内存中创建一个对象，并用this指向它，在此函数中，可以给this进行初始化设置。
  3. 最终构造函数隐式返回this给调用者

构造函数中的 return

  1. 如果没有返回值，则会自动把this隐式的返回给外部调用者
  2. 如果是返回其他对象，那么返回者就是此对象
  3. 如果返回的是基本（原始）类型，则返回的值依然是this

换句话说，带有对象的 return 返回该对象，在所有其他情况下返回 this。

// 模拟一个类（创建一个构造函数（构造器））
// Person就是类名，（首字母大写）
function Person(name,age){
  this.name = name
  this.age = age
  this.run = function(){
    // this指向当前调用者。
    return this.name +'在跑步'
  }

  // 会自动把this隐式的返回给外部调用者
  // return this; // 默认会显示返回  一般不需要写返回值，会默认返回当前对象this
  // return {a:2} // 返回 {a:2}
  // return 123; // 若原始类型，则还是返回this
}

this关键字详解

JavaScript中的this指向问题，有时候会让人难以捉摸，随着学习的深入，我们可以逐渐了解 现在我们需要掌握函数内部的this几个特点

1. 函数在定义的时候this指向是不确定的，不能看它定义的位置，要看this所在的执行环境。
2. 一般函数直接执行，内部this指向全局window
3. 函数作为一个对象的方法，被该对象所调用，那么this指向的是该对象

总结一句话，this永远指向最后的调用者，若没有调用者一般都是指向全局对象window

示例代码：

<script>
        // 'use strict';
        var name = '全猪猪'
         // 一句话：this永远指向最后的调用者
        //  非严格模式下，如果没有对象去调用方法，则this=>window
        //  严格模式下，如果没有对象去调用方法，则this=>undefined

        function Person(name){
            this.name = name;
            this.getName = function(){
                console.log('my name is ' + this.name)
            }
        }

        var p1 = new Person('李猪猪')
        p1.getName(); // 李猪猪

        var getName = p1.getName;
        getName(); // 全猪猪


        var obj = {
            name: "虎虎"
        }
        // 让obj去伪造getName中的this
        p1.getName.call(obj)

    </script>

构造函数当做普通函数调用

function Person(){
  console.log(this); // 普通调用this指向window, new调用指向当前创建的对象
}

Person();

new Person();
  • 当普通函数来调用this指向window
  • 当构造函数来调用this指向当前创建的对象

假设我们定义了一个构造函数，限制只能new来调用，那么，如何防止用户直接当做普通函数来调用呢？

解决办法：用 instanceOf 关键字。

因为一个对象如果是通过构造函数创建出来的，那么这个对象一定是构造函数的实例，通过instanceOf 来判断即可。

      function Person(){
        if(this instanceof Person){
            console.log('构造函数调用');
        }else {
            console.log('普通函数调用');
        }
      }

      Person();

      new Person();

如果发现用户是直接当做普通函数调用，我们在内部帮其new操作即可。

function Person(name, age) {
  if (!(this instanceof Person)) {
    return new Person(name, age)
  }
  this.name = name;
  this.age = age;
}

console.log( Person('王老五',55) );

console.log( new Person('老六',66) );

js基本包装类型[了解]

js除了基本数据类型和引用数据类型外，JavaScript还提供了三种基本包装对象：Number、Boolean和String。

let str = 'abc' 
typeof str; // 'string'

let str2 = new String('abc') 
typeof str; // 'object'
str === str2; // false

虽然包装对象看上去和原始值一模一样,用typeof查看他们的类型已经变为object了！所以，包装对象和原始值用===比较会返回false，因为其类型不一样

基本(原始)类型和基本包装类型的最大区别就是生存期不同，在代码执行后就会销毁实例。

  • 引用类型：使用new操作符创建的引用类型实例，在执行时，离开当前作用域之前一直都保存在内存中，

  • 基本包装类型：给基本类型当做对象添加属性和方法后，会立即立即销毁。

    基本包装类意味着我们不能在运行时为对象添加自定义属性及方法，因为它们执行完后会立即销毁。

let str = '张三'
str.age = 18 // 即 var str = new String('张三')，str.age = 18; 执行完毕后立即销毁
console.log(str.age) // undefined

第二行代码执行完后立即销毁，第三行代码再次创建String对象发现没有该age属性。

原型对象

为什么要学习原型对象

先来看下一下代码：

    function Person(name){
        this.name = name
        this.hobby = function(){
            console.log('开二手法拉利');
        }
    }

    var p1 = new Person('老六')
    var p2 = new Person('老王')

    // 对于对象、数组、函数名的比较，都是比较他们在堆中的内存地址
    console.log( p1.hobby === p2.hobby ); // false ， 说明占用不同的堆内存

上面的值为false。 由此可知，仅通过构造函数创建的对象会浪费内存空间。

原因是每个对象都是在自身（this）的内存空间中单独存储一个公有方法hobby，我们应该让这些公有的方法让每个对象共享。

我们目标是希望把这些公共的方法或属性抽离出来，从而节省内存空间。

内存图解：

解决办法： 借助原型对象。

什么是原型对象

三个特点（务必记住）：

  • 每个构造函数都会有一个prototype属性，其值是一个对象,此对象就是原型对象。
  • 通过构造函数new出来的实例对象，其身上会有一个__proto__属性，其值也是原型对象。
  • 原型对象会有个constructor属性，指向构造函数本身
function Person(name){
  this.name = name
}

Person.prototype.hobby = function hobby(){
  console.log('都爱打麻将');
}

var p1 = new Person('老六')
var p2 = new Person('老王')
console.log( p1.hobby === p2.hobby ); // true, 共享同一块内存 

console.log(Person.prototype === p1.__proto__); // true ,指向同一个原型对象

内存图解：

可见，原型对象优点：

  • 抽离对象的公有属性或方法，解决内存空间浪费问题。
  • 用于对象的继承，实现代码的复用。

原型链

  • 原型对象本身也有自己的原型，原型链就是各种原型形成的一个链式结构。
  • 构造函数的prototype属性指向其原型对象，原型对象也会有个__proto__属性，指向Object.prototype原型。
  • Object.prototype对象同样也有个__proto__属性，其值为null,此时就是原型链的终点。
function Person(name){
  this.name = name
}

Person.prototype.hobby = function hobby(){
  console.log('打麻将');
}

var p1 = new Person('老六')

console.log(Person.prototype === p1.__proto__); // true ,指向同一个原型对象

console.log(Person.prototype.__proto__ === Object.prototype); // true

console.log(Object.prototype.__proto__); // null, 原型链终点

JS原型链终极图解

自定义的构造函数会有原型对象，其实JS中内置的构造函数如String、Array、Number、Function、RegExp、Date、Error、Function、Boolean、Object，他们也有自己的原型对象。如String.prototype.__proto__指向Object.prototype。 Object.prototype.__proto__值为null，即到了原型链的顶端

  • 数组对象之所以能调用push、pop等相关方法，这些方法就是定义在Array.prototype原型中。
  • 字符串对象之所以能调用indexOf、substr等相关方法，这些方法就是定义在String.prototype原型中。
  • 函数对象之所以能调用call，apply，bind等相关方法，这些方法就是定义在Function.prototype原型中。
  • 日期对象之所以能调用getFullyear、getMonth等相关方法，这些方法就是定义在Date.prototype原型中。

如他们的原型上找不到，就会就会顺着__proto__属性去Object.prototype中找。

说明：

  • Math也可以叫做静态类，不能new操作，因为没有构造函数。调用方法时候，格式为：Math.方法名(),如调用random方法，Math.random(); Math.ceil() ;
  • 其中有个Object类，Object类是所有类的父类（基类）。它是整个原型链的顶端。

DOM节点对象和事件对象他们也有自己继承链。

  • DOM节点对象继承链： HTMLButtonElement -> HTMLElement -> Element -> Node -> EventTarget -> Object
  • 事件对象继承链： PointerEvent -> MouseEvent -> UIEvent -> Event -> Object
// Object(超类、基类)
Object.prototype.sayHello = function(){
    console.log('你好')
}
document.getElementById('btn').onclick = function(event){
    this.sayHello();
}

原型链中查找标识符的原则

所谓的标识符就是变量名和方法名。

当对象找一个标识符时，会优先在自己的内存空间进行查找，如果找不到，则顺着原型对象__proto__属性找到他们的原型，最终会找到Object.prototype内存空间中，找到则返回结果。

寻找的规则：就近原则。 儿子(自身) ->爸爸 -> 爷爷 -> null

如果在整个原型链中属性找不到则返回undefined、方法找不到则报错！

原型链中相关API

  • hasOwnProperty：判断自身内存空间中是否存在某个属性
  • isPrototypeOf:判断某个对象是否是是一个对象的原型
  • instanceof:判断某个对象是否是其原型链上的实例
  • for-in: 用来循环对象自身空间的属性及原型中的属性
function Person(name){
            this.name = name;
        }

        Person.prototype.email = '11111@qq.com'

        var p1 = new Person('桐哥')
        // hasOwnProperty：判断自身内存空间中是否存在某个属性
        console.log(p1.hasOwnProperty('name')); // true
        console.log(p1.hasOwnProperty('email')); // false

        // isPrototypeOf:判断某个对象是否是是一个对象的原型
        console.log( Person.prototype.isPrototypeOf(p1) )
        console.log( Object.prototype.isPrototypeOf(p1) )

        // instanceof:判断某个对象是否是其原型链上的实例
        console.log(p1 instanceof Person); // true
        console.log(p1 instanceof Object); // true

        // for-in: 用来循环对象自身空间的属性及原型中的属性
        for(var k in p1){
            console.log(k,':',p1[k]);

valueOf和toString方法

这两个方法来自于Object.prototype原型对象中。

  • valueOf() 方法返回指定对象的原始值。
  • toString() 方法返回一个表示该对象的字符串。

不同对象调用valueOf，返回值也不同。

对象 返回值
Array 返回数组对象本身。
Boolean 布尔值。
Date 返回的是从 1970 年 1 月 1 日到现在的毫秒值
Function 函数本身。
Number 数字值。
Object 对象本身。这是默认情况。
String 字符串值。 
var a = 1;
console.log(a.valueOf()) // 1

var b = 'hello';
console.log(b.valueOf()) // 'hello'

var obj = {name:'张三'}
console.log(obj.valueOf()) // [object Object]

toString():返回对象的字符串表示：

var a = 1;
console.log(a.toString()) // '1'

var b = 'hello';
console.log(b.toString()) // 'hello'

var obj = {name:'张三'}
console.log(obj.toString()) // '[object Object]'

构造函数中重写toString()

重写前：

function Dog(name, color) {
    this.name = name;
    this.color = color;
}

var theDog = new Dog("旺财", "red");
console.log(theDog.toString()); // [object Object]

上面的toString实际是从Object.prototype中继承的。

重写后：

Dog.prototype.toString = function () {
 return "狗名：" + this.name +  " , 颜色： " + this.color
}
console.log(theDog.toString()); // '狗名：旺财 , 颜色： red'

使用 toString() 检测对象精确类型

Object.prototype.toString.call([]) // '[object Array]'
Object.prototype.toString.call({}) // '[object Object]' 即原始对象 plainObject
Object.prototype.toString.call(new Date()) // '[object Date]'
Object.prototype.toString.call(1) // '[object Number]'
Object.prototype.toString.call(true) // '[object Boolean]'
Object.prototype.toString.call('123') // '[object String]'
Object.prototype.toString.call(undefined) // '[object Undefined]'
Object.prototype.toString.call(null) // '[object Null]'

当然上面call换成apply也行

实现功能：封装一个函数，判断是否是数组！

不能借助内置的Array.isArray()函数

// 需求：封装一个方法，判断是否是数组
function isArray(data){
    var toString = Object.prototype.toString;
    if(toString.apply(data) === '[object Array]'){
        return true;
    }
    return false;
}

console.log(  isArray(arr) ); // true
console.log(  isArray(obj) ); // false

注意：toString和valueOf两个方法可被重写,以实现更适合自身的功能需要 

var obj = {
    a: 1,
    valueOf: function () {
        console.log('重写valueOf')
    },
    toString: function () {
        console.log('重写toString')
    }
}

obj.valueOf() // 重写valueOf
obj.toString() // 重写toString

为了防止toString被重写(覆盖)的情况，更严谨的做法就是获取原始的toString来实现类型判断：

// 获取原始的toString
var toString = Object.prototype.toString;

toString.call(1) // '[object Number]'
toString.call([]) // '[object Array]'
toString.call({}) // '[object Object]'

静态属性[了解]

通过构造函数名直接添加的属性或方法，可以称之为静态属性或静态方法。

访问静态属性的时候也只能通过构造函数名类访问！不能通过对象来访问，因为它不属于某个对象。

作用：设置一些通用的工具函数。

    <script>
        // Person是构造函数的名字
        function Person(name){
            this.name = name;
        }

        // 添加静态属性
        Person.version = '1.0.0';
        // 添加静态方法
        Person.getInfo = function(){
            console.log(this); // 
            console.log('我是静态方法getInfo')
        }

        // 静态的属性或方法只能用构造函数名去调用
        console.dir(Person.version)
        Person.getInfo();

        // var p = new Person('桐哥')
        // console.log( p.version ); // undefined
        // p.getInfo(); 报错

        // 作用：一般都是通过静态方法给构造函数绑定一些工具函数。
        Person.isArray = function(data){
            var toString = Object.prototype.toString;
            if(toString.call(data) === '[object Array]'){
                return true;
            }
            return false;
        }


        console.log( Person.isArray([1,2,3]) ); // true
        console.log( Person.isArray({}) ); // false


    </script>

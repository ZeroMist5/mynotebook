08-作用域&调用栈(call stack)

作用域

什么是作用域

作用域: 用来描述变量或函数有限的访问范围。

通俗一点就是某个变量哪里可访问哪里不可访问。

作用域

作用域分类：

  • 全局作用域: 非函数内的区域就是全局作用域。
  • 局部（函数）作用域: 在函数内部就是局部（函数）作用域
  • 块级作用域（ES6新增）ECMAScript6（ES6-let）

全局作用域（Global Scope）

  • 在全局作用域定义的变量/函数我们称之为是全局变量/函数，它在任何地方都可以访问。 
var age = 18; // 全局变量
console.log(age); // 18

function  getAge(){
  console.log('我今年 '+ age + '岁')
}
getAge();

局部作用域（Local Scope）

  • 在函数内部就是局部（函数）作用域
  • 每调用一次函数就会创建一个新的函数作用域，它们之间是相互独立的。
  • 函数内部声明的变量我们称之为局部变量，它只能在所定义的函数内部可访问，在全局中是无法访问的。

练习：

var age = 20; // 全局变量age
var email = '110@qq.com' // 全局变量email
function foo() {
    var age = 30;  // 局部变量
    var address = '汕头'; // 局部变量
    console.log(age); // 30
    console.log(email); // '110@qq.com'
}
foo();
console.log(age); // 20
console.log(email); // '110@qq.com'
console.log(address); // 报错

注意：全局中默认有个特殊的变量name,值为空'',平时的测试代码不要被name的默认值给影响到。

注意：在函数内若定义一个未使用var声明的变量，则执行此函数时会覆盖全局内同名的变量，这种代码是不规范的，不建议书写，一定要避免，否则对我们的代码造成意想不到的效果，也不利于代码调试。

示例：

var myname = "大锤";
function foo() {
    myname = "小锤"
}
foo();
console.log(name)

有个网站可以检测js代码是否符合规范， http://jshint.com/

作用域链（scope chain）

作用域链：就是变量的查找规则（就近原则）。

要注意作用域和作用域链的区别

  • 作用域描述的是变量作用的范围。

  • 而作用域链决定了变量的查找规则（就近原则，由内往外层找）。

示例代码：

var age = '100';
function foo(){
    console.log(age);     // ?
    function bar(){
        var age = '18';     
        console.log(age);     // ?
        function par(){
            console.log(age);     // ?
        }
        par();
    }
    bar();
}
foo();

说明：

  • 只有函数才可以形成作用域结构。如果没有函数那就只存在一个域，就是window全局作用域。

  • 作用域链中查找变量的原则：就近原则。

    就近原则：访问变量，会优先在自身的作用域查找，若没有找到，会继续往上一级作用域中去找，直到找到全局作用域，若找到就使用，找不到就报错（引用错误，这个变量没有定义 xxxx not a defined）

全局对象window

  • 在全局作用域中有一个全局对象window（代表的是一个浏览器的窗口对象，由浏览器创建），可以直接使用。
  • 在全局作用域中，所有使用var声明的变量都会作为window对象的属性，所有声明的函数都会作为 window 对象的方法保存。 
        // 在全局作用域中有一个特殊的对象，叫window，一般称为window全局对象
        console.log(window)
        // 在全局中this和window是同一个对象，this指向window的同一个内存地址
        console.log(this)

        // 对象是比较其内存地址
        console.log(this === window); // true

        // 在全局作用域中使用var声明的变量会自动成为window对象的一个全局属性,
        // 声明的函数，其函数名也会成为window的一个全局属性
        var aaa = 111;
        console.log(aaa)
        // console.log(window.aaa)
        // console.log(this.aaa)

        function foo() {
            console.log('foo函数调用了')
        }

        foo(); // 推荐此种方式

调用栈（Call Stack）

什么是调用栈

  • 调用栈（Call Stack）是js解释器追踪函数执行流程的一种机制。
  • 当执行环境中调用了多个函数时，通过这种机制，我们能够追踪到哪个函数正在执行，执行的函数体中又调用了哪个函数,可知道代码的执行流程，也利于bug的查找。

示例：

函数A中调用函数B,函数B中调用了函数C。

function A(){
    console.log('A')
    B();
}

function B(){
    console.log('B')
    C();
}

function C(){
    console.log('C')
}

A();

上述代码在执行调用栈中调用过程如下：

栈结构特点： 先进后出，后进先出。

  1. 全局代码（script）首先产生一个全局执行上下文（全局环境-main)压入调用栈的底部
  2. 每调用一个函数，该函数就会压入到调用栈的顶部
  3. 当某个函数执行完毕之后，此函数就会出栈。其函数内部的局部变量都会被销毁。
  4. 当所有的函数执行完毕之后，只剩下全局上下文了。
  5. 全局上下文只有用户关闭浏览器才会被释放

断点调试

介绍：

断点调试是指自己在程序的某一行设置一个断点，调试时，程序运行到这一行就会停住，然后你可以一步一步往下调试，调试过程中可以看各个变量当前的值，出错的话，调试到出错的代码行即显示错误，停下。

多数用于分析某段js代码或一个js库（如：Vue库）的源代码是如何一步步执行的，那么就可以使用断点调试进行代码追踪分析。

代码中进行断点调试：

debugger; // 打断点调试，代码执行这里，会直接停下。通过按钮逐步往下执行

也可以通过浏览器进行断点调试

浏览器中按F12-->sources-->找到需要调试的文件-->在程序的某一行点击设置断点

注意：

  • js引擎是单线程：意味着同一时刻只能做一件事件。即js种只有一个call Stack(执行栈)
  • 当分配的调用栈空间被占满时，会引发“堆栈溢出（RangeError: Maximum call stack size exceeded）”错误

函数内部this的指向问题

一句话总结：this永远都是指向最后的调用者。

  • 如果是普通对象调用函数,this一般是指向当前调用者。
  • 如果是在全局直接调用一个函数，this一般都是指向window全局对象。
  • this到底是谁不要看其定义在哪里，要看指向的环境。
var age = 19;
var obj = {
    age:18,
    getAge:function(){
        console.log('my age is:' + this.age)
    }
}

obj.getAge();

函数内部arguments属性

arguments是函数内部的特殊属性。

  • arguments作用：获取所有的实参。
  • arguments是一个伪数组。所谓伪数组就类似于真数组一样，可通过下标获取元素和通过length属性获取元素个数。也可for循环访问元素，但是不能调用真数组的方法（如：push）。
    function foo(){
        console.log(arguments.length); // 5 代表传递了5个实参
    }

    foo(1,2,3,4,5);
  • 案例：求任意个数数字的和
 function sum(){
        var sum = 0;
        for(var i = 0; i < arguments.length; i++ ){
            sum += arguments[i]
        }
        return sum;

    }

    console.log(  sum(10,20,30,40) ); // 100

arguments应用场景： 可以实现不同的传参个数，实现不同的业务逻辑。

如： Jquery插件中有css方法：

css('color'); // 一个参数，获取节点颜色
css('color','red'); // 两个参数，设置节点颜色

扩展：将伪数组转换为真数组

function sum(){
  var arg2 = Array.from(arguments); // 伪数组转化为真数组
  arg3.push(4)
}

sum(1,2,3)

获取函数定义的形参个数:

函数名.length

function add(num1,num2,num3){
    return num1 + num2 + num3;
}
// 函数名.length: 获取函数定义的形参个数；
console.log( add.length ); // 3

事件对象event

什么是事件对象？

当一个事件被触发时，会创建一个对象(Event Object), 这个对象里面包含了与该事件相关的属性或方法。

作用：获取所在页面中的xy坐标，知道按了什么键等。。。

不同的事件，该对象会作为第一个参数传递给事件的处理函数。

btnEle.onclick = function(event){
    console.log(event)
}

由于不同浏览器事件对象会有一点差异性，下面写法可兼容所有浏览器。

btn.onclick=function(event){
    // window.event  IE浏览器事件对象(固定写法),兼容 IE9以下
    // event    w3c标准浏览器对象(edge)
      event = event || window.event    
 }

示例：

<body>
    <button id="btn">button1</button>
    <!-- 手动传递一个事件对象event,固定写法 -->
    <button onclick="hello(event,123)">button2</button>
    <script>
        document.getElementById('btn').onclick = function(ev){
            event = ev || window.event
            console.log(event); // 事件对象
        }


        function hello(event,msg){
            console.log('hello',event,msg)
        }
    </script>
</body>

事件对象属性和方法

不同浏览器的事件对象属性也有差异性：

W3C事件对象常用属性:

  • type：用于获取事件类型如单击事件为click,失去焦点为blur
  • KeyCode或which： 获取键盘上面的键码值
  • target：事件发生的目标（主体）元素。
  • pageX: 获取在页面中x坐标
  • pageY: 获取在页面中y坐标
  • stopPropagation()阻止事件冒泡
  • preventDefault()阻止事件默认行为

IE事件对象常用属性:

  • type用于获取事件类型
  • clientX: 获取在客户端浏览器中x坐标
  • clientY: 获取在客户端浏览器中y坐标
  • cancelBubble阻止事件冒泡
  • returnValue阻止事件默认行为

事件模型

在JavaScript中事件模型总共可以分为三类：

  • DOM 0级模型（原始事件模型）
  • IE事件模型（了解）
  • DOM 2级模型（W3C标准模型）

不同的事件模型，绑定事件的方式也不一样，事件流也不一样。

DOM0事件模型

DOM0事件模型又称为原始事件模型

绑定事件有以下两种方式

  • HTML代码中直接绑定(行内绑定方式也叫做静态绑定):
<input type="button" onclick="f1()" value="按钮">
  • 通过JS代码绑定
var btn = document.getElementById('btn');
btn.onclick = f1;    //动态绑定方式
btn.onclick = null;    //移除绑定的单击事件

DOM0级的好处：大部分浏览器都支持以上两种的写法，即兼容性最好

示例：

<body>
    <button id="btn1">戳我</button>
    <button id="btn2">不戳</button>
    <script>
        var btn1 = document.getElementById('btn1')
        var btn2 = document.getElementById('btn2')

        function fn(){
            alert('好痛')
        }

        // 给btn1绑定单击事件
        // btn1.onclick = 匿名函数/有名函数
        // btn1.onclick = fn;
        btn1.onclick = function (){
            alert('好痛')
        }

        // 单击btn2解绑（移除）btn1的单击事件
        btn2.onclick = function(){
            btn1.onclick = null;
        }
    </script>
</body>

DOM2事件模型

介绍

  • 属于W3C标准模型，现代标准浏览器(除低版本IE（5，6,7,8）)都支持该模型。

DOM2事件模型绑定事件的方式:

事件绑定监听函数的方式如下:

dom对象.addEventListener(eventType, handler, useCapture)

事件移除监听函数的方式如下:

dom对象.removeEventListener(eventType, handler, useCapture)

注意：如果事件处理函数handler是有名函数，则可以通过名字来移除，匿名函数无法移除

参数说明：

  • eventType 指定事件类型(不要加前缀on)
  • handler事件的处理函数
  • useCapture可选。布尔值，指定事件是否在捕获或冒泡阶段执行。
    • true 事件监听函数在捕获阶段执行
    • false 默认，事件监听函数在冒泡阶段执行

核心代码如下：

var btn = document.getElementById('btn');
btn.addEventListener('click', handler);   //绑定
btn.removeEventListener('click', handler);  //移除

<body>
    <button id="btn1">戳我</button>
    <button id="btn2">不戳</button>
    <script>
        var btn1 = document.getElementById('btn1')
        var btn2 = document.getElementById('btn2')


        function fn(event){
            alert('好痛')
        }

        // DOM2，事件名不可以加前缀on
        btn1.addEventListener('click',fn)

        // 单击btn2要解绑btn1的单击事件
        btn2.addEventListener('click',function(){
            // 解绑
            btn1.removeEventListener('click',fn)
        })
    </script>
</body>

IE事件模型（了解）

绑定事件和移除事件（仅适用于IE浏览器）:

绑定事件方式如:

dom对象.attachEvent(eventType, handler);

解绑事件方式如下

dom对象.detachEvent(eventType, handler);

参数说明:

​ eventType指定事件类型(注意加前缀on)

​ handler是事件处理函数

示例：

<body>
    <button id="btn1">戳我</button>
    <button id="btn2">不戳</button>
    <script>
        var btn1 = document.getElementById('btn1')
        var btn2 = document.getElementById('btn2')


        function fn(event){
            alert('好痛')
        }

        // IE事件模型(了解，IE11不行，以下都可以)
        btn1.attachEvent('onclick',fn)

        btn2.attachEvent('onclick',function(){
            // 解绑btn1单击事件
            btn1.detachEvent('onclick',fn)
        })

    </script>
</body>

事件流

  1. 什么是事件流？

事件流：即当一个事件发生时，事件会在特点的节点之间按照特定的顺序传播。该事件的传播过程就叫做事件流

  1. 事件流的三个阶段

  2. 事件目标阶段(触发自身元素事件)：触发当前自身的事件。

  3. 事件捕获阶段：事件开始由顶层元素触发，然后逐级向下传播，直到目标的元素。
  4. 事件冒泡阶段：事件由具体的元素先接收，然后逐级向上传播，直到不具体的元素。

以上面的图解为例，如果给上图的四个元素都绑定单击事件，由图可以很清晰地看出事件的触发顺序为：

先捕获阶段——》目标元素阶段——》最后冒泡阶段。

  • DOM0事件模型：目标阶段和冒泡阶段
  • DOM2事件模型：捕获阶段、目标阶段和冒泡阶段
  • IE事件模型：目标阶段和冒泡阶段

事件流顺序：捕获（从上往下）=》目标（自身）=》冒泡（从下往上）

并不是所有的事件都有冒泡：常见的有click,mouseover会有冒泡

捕获和冒泡的作用：

  • 捕获在实际项目开发中并不多见，但是冒泡有特殊作用（实现事件委托，提高事件绑定的性能）

事件冒泡

事件冒泡

介绍:

事件冒泡：事件由目标元素开始接收，然后逐级冒泡到祖先元素。

冒泡原理:

根据用户点击的xy坐标来判断是否在父容器的管辖区域内，如果在，则获取事件对象的target属性确定事件发生的主体，如果事件主体绑定了冒泡事件，这个事件就会逐级向上传播。

有冒泡的事件：click,mouseover和mouseout 。

其他事件都不会冒泡

mouseenter（鼠标悬浮）和mouseleave（鼠标离开）:不会冒泡

如何阻止事件冒泡:

可通过事件对象相应的方法实现，进而阻止向上传播

e.stopPropagation(); // w3c事件对象
e.cancelBubble = true; // IE事件对象

如何判断是w3c事件对象? 通过对象的方法名进行判断，

// 对象的方法名可以作为对象的属性进行判断，从而知道是什么浏览器
if ( event.stopPropagation ){
    event.stopPropagation();    // w3c
}else{
    event.cancelBubble = true;    // IE
}

事件委托（事件代理）

介绍：

事件委托也叫事件代理，使用事件委托技术能避免对特定的每个子节点添加事件监听,相反,事件监听器是被添加在他们的祖先元素上，这样绑定事件的性能更高，就不需要循环的给每个子节点都绑定事件。

通俗的说：就是某个事件本来该是自己完成的，但是自己不干，就叫父亲来干，这就叫事件委托。

注意：事件委托的原理是事件冒泡。在事件委托中不能阻止事件冒泡机制，否则事件委托不成功。

传统多元素绑定事件方式

传统的做法给如下的li标签都绑定单击事件：

<ul id="list">
    <li>111</li>
    <li>222</li>
    <li>333</li>
    <li>444</li>
    <li>555</li>
    <li>李同学</li>
</ul>
<button>新增同学<button>

如果我们需要给上面的所有的li绑定单击事件我们一般会这么做，如下：

var lis = document.getElementsByTagName('li');
for(var i = 0,length = lis.length; i < length;i++){
      lis[i].onclick=function(event){ 
         console(this.innerHTML);        //打印当前元素的标签中的内容
      }
}

缺点：

以上代码有个不好的地方,就是当给ul中动态添加li节点时,添加后的li是没有onclick事件的,这在开发中是不不允许的。且上面的写法是循环给每个li都绑定onclick事件,一旦li节点比较多的时候，这时候性能非常差。

怎么解决？ 通过事件委托，委托给li的祖先元素，如委托给ul即可。

通过委托方式给多元素绑定事件

  • 事件委托并不是每个子节点单独设置事件监听。
  • 相反是把事件绑定在其祖先节点上。

实现委托的步骤：

  • 给ul设置单击事件，因为点击ul内的li，根据冒泡原理，事件会冒泡到ul上，ul有注册事件，就会触发其事件。
  • 然后利用事件对象的target属性来找到当前点击的目标元素li，并处理相应业务逻辑
var listEle = document.getElementById('list');
// 委托给父元素ul
listEle.onclick = function(event){
    if(event.target.localName === 'li'){
        // 处理相应业务逻辑
        console.log('单击了li')
    }
}

事件委托优势：

  • 事件只绑了一次，提高了绑定事件的性能

什么时候使用事件委托呢

  • 需要给列表元素绑定事件时。
  • 让动态新增的元素也有事件

示例：

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        li span {
            cursor: pointer;
        }
    </style>
</head>

<body>
    <!-- autofocus自动获取焦点 -->
    <input type="text" id="username" placeholder="请输入用户名"> <button id="btn">添加</button>
    <ul id="list">
        <li>陈安烂 <span title="删除"> × </span> </li>
        <li>肖林森 <span title="删除"> × </span></li>
    </ul>
    <script>
        var btn = document.getElementById('btn')
        var user = document.getElementById('username')
        var list = document.getElementById('list')
        var spans = document.getElementsByTagName('span')

        btn.onclick = function () {
            // 1. 获取内容
            var username = user.value;
            user.value = '';
            // 2. 创建li标签，并追加到ul
            var li = document.createElement('li');
            li.innerHTML = username + '<span title="删除"> × </span>'
            list.appendChild(li)
        }


        // 采用委托方式绑定，将span的单击事件委托给UL。
        list.onclick = function (event) {
            // 判断点击是span，才删除li元素
            if (event.target.localName === 'span') {
                // 删除li
                var liNode = event.target.parentNode; // li
                var ulNode = event.target.parentNode.parentNode; // ul
                ulNode.removeChild(liNode)
            }
        }
    </script>
</body>

</html>

默认行为

什么是默认行为，即标签本身就有的行为特征！

哪些标签有默认行为呢？，常见的有以下两个标签：

  • 链接 <a href="./index.php">点我</a> ，默认行为会往属性href指定的地址跳转
  • 表单内点击提交按钮（type=submit） ， 默认行为会往form表单属性action指定的地址跳转

这里注意 和 的区别：

  • type=submit :点击默认提交表单，即有默认行为。等同于一对Button标签
  • type=button:就是一个普通的按钮，点击不会提交表单，需要给其绑定事件才可以提交表单。

阻止默认行为有两种方式：

  • 通过事件对象

     event.preventDefault(); // w3c
 event.returnValue = false; // IE

  • 或者给事件处理函数返回一个false。 

<!-- action:跳转的地址  -->
<form action="/login" onsubmit="return false;"> 
  用户名： <input type="text" name="username">
 <input type="submit" value="提交">
</form>

<a href="http://baidu.com" onclick="return false">百度</a>

示例：

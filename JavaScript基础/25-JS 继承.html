25-JavaScript继承

对象的继承

在JavaScript中实现对象的继承常用的方式：

  • 通过call和apply实现伪造（冒充）继承 [单独使用存在问题]
  • 通过原型对象实现继承 [单独使用存在问题]
  • [重点]（组合继承）通过伪造+原型对象实现完整继承 【结合上面两者，形成互补】
  • 寄生继承：修补组合继承执行父类构造两次的问题。

通过call或apply方法实现伪造（冒充或构造函数）继承

call和apply方法介绍

  1. call和apply的作用：

调用函数并改变函数内this的指向：

使用语法：

functionName.call(obj,实参数1，实参数2...)     
functionName.apply(obj, [ 参数1，参数2..... ] )

通过call或apply方法实现伪造继承

  1. 通过call或者apply继承的思想：

在子类的构造函数中，使用call或apply去伪造（充当）父类构造函数中的this，把父类构造函数中的属性添加到子类对象的自身空间中。

核心代码：

function 子类(){
   //使用当前对象this去伪造父类构造函数内的this
   父类.apply(this); 
}
  1. 参考代码：

// 父类构造函数
function Parent(name, age) {
    this.name = name
    this.age = age
}

// 父类原型对象添加方法
Parent.prototype.getName = function () {
    console.log('my name is ' + this.name)
}

// 子类构造函数
function Child(name, age, email) {
    // 核心：用当前对象的this去伪造父类构造函数中的this
    Parent.call(this, name, age)
    this.email = email
}

var c1 = new Child('张三', 13, 'zs@qq.com')
console.log(c1)
console.log(c1.name)
c1.getName()

// 伪造继承优点：可以把父类构造函数中的this定义属性设置到子类构造函数对象自身空间中。
// 局限性（缺点）：无法继承父类原型对象中的属性或方法。
  1. 通过call或apply伪造继承小结：

通过call或apply只能会把父类构造函数中定义的属性复制一份到子类对象自身空间中，但是无法继承到父类原型对象上面的属性。

解决办法： 通过原型继承来实现，可以继承父类原型对象中的属性或方法。

通过原型对象继承

回顾三个定理:

  • 构造函数会有一个prototype属性，值指向原型对象
  • 原型对象会有个constructor属性，值指向构造函数本身

  • 通过构造函数new出来的对象，都会有一个__proto__,值也是指向原型对象

  • 原型对象继承的核心思想： 用父类对象重写子类原型对象。核心代码如下

    重写=覆盖

子类.prototype.hobby = function；  // 在原有对象空间中追加
子类.prototype = new 父类()；  // 直接重写
  1. 参考代码
      // 父类构造函数
      function Parent(name, age) {
        this.name = name
        this.age = age
      }

      // 父类原型对象添加方法
      Parent.prototype.getName = function () {
        console.log('my name is ' + this.name)
      }

      // 子类构造函数
      function Child(name, age, email) {
        this.email = email
      }

      //   核心：子类.prototype = new 父类();
      Child.prototype = new Parent('无名', 0)

      // 给子类原型添加方法
      Child.prototype.getEmail = function () {
        console.log('email is ' + this.email)
      }

      var c1 = new Child('张三', 13, 'zs@qq.com')
      console.log(c1)
      c1.getEmail()
      c1.getName()

      //   原型对象继承优点：可以继承父类原型对象中的属性。
      //      缺点：无法把父类构造函数中的属性设置到子类对象自身空间中
  1. 通过上面原型继承带来的问题：

当子类的某个对象对引用类型属性color添加一个yellow的颜色，实质上，我们是把此值添加在子类原型对象中的color属性中，这样会造成所有子类对象对此属性的读取，这是我们不允许的。

我们希望，每个子类自身空间中都独有一份color属性，则当子类对象修改color属性时只会影响当前对象，对其他对象没有任何影响。通过伪造来解决

怎么解决？

答：通过伪造继承+原型继承组合起来，就可以解决伪造继承和原型继承单独使用带来的问题。

通过伪造和原型对象组合继承

  1. 回顾伪造继承和原型继承分别带来的问题

  2. 单独伪造继承带来的问题：只能将父类构造函数中的属性放到子类对象的自身空间中，但是无法继承父类原型对象上面的属性。

  3. 单独原型继承带来的问题：子类对象只能继承父类原型对象上面的属性，但是无法把继承父类构造函数中的属性放到自身空间

  4. 通过伪造和原型对象组合继承的核心思想：

  5. 通过伪造继承,把父类构造函数中的属性加到到子类对象的自身空间中

  6. 通过原型继承,继承到父类原型上面的属性/方法 最后把他们两者组合起来即可（形成互补）

如：

伪造继承：它会喝酒，但不会抽烟

原型继承：不会喝酒，但会抽烟

组合起来，可实现又会抽烟又会抽烟。 （ps:少抽烟喝酒，年轻人身体最重要! ）

  1. 代码如下：
// 父类构造函数
      function Parent(name, age) {
        console.log('Parent构造函数执行')
        this.name = name
        this.age = age
      }

      // 父类原型对象添加方法
      Parent.prototype.getName = function () {
        console.log('my name is ' + this.name)
      }

      // 子类构造函数
      function Child(name, age, email) {
        // 伪造继承核心（目的：把父类构造函数中的属性设置到子类对象自身空间中）：
        Parent.call(this, name, age)
        this.email = email
      }

      // 原型继承核心（目的：把父类原型对象中的属性或方法继承过来）：子类.prototype = new 父类();
      //   Child.prototype = new Parent('无名', 0)
      Child.prototype = Parent.prototype

      // 给子类原型添加方法
      Child.prototype.getEmail = function () {
        console.log('email is ' + this.email)
      }

      var c1 = new Child('张三', 13, 'zs@qq.com')
      var c2 = new Child('威威', 19, 'ww@qq.com')

      //   组合继承缺点：执行了父类构造函数两次，浪费性能。
      //   或这么写Child.prototype = Parent.prototype：缺点：如果父类原型有个引用属性，那么所有的
      // 子类对象的修改读取都会受到影响

寄生继承

上面的组合继承（伪造继承 + 原型继承）： 将父类构造函数执行了两次，浪费性能。

优势： 规避了父类构造函数执行两次的问题。

寄生继承核心：让子类原型被空函数实例重写 通过一个空函数的原型继承父类原型。

// 继承路径： 子类原型 -> 空函数实例（对象） -> 父类原型
function F(){}
Child.prototype = new F();
F.prototype = Parent.prototype

完整代码：

      // 父类构造函数
      function Parent(name, age) {
        // console.log('Parent构造函数执行')
        this.name = name
        this.age = age
        this.hobby = ['睡觉']
      }

      // 父类原型对象添加方法
      Parent.prototype.getName = function () {
        console.log('my name is ' + this.name)
      }

      // 子类构造函数
      function Child(name, age, email) {
        // 伪造继承核心（目的：把父类构造函数中的属性设置到子类对象自身空间中）：
        Parent.call(this, name, age)
        this.email = email
      }

      // 寄生继承路径： 空函数原型  -> 父类原型 -> 子类原型被空函数实例重写
      function F() {}
      F.prototype = Parent.prototype

      Child.prototype = new F()
      //  修正子类原型对象中的constructor属性，保证原型链的完整性，让子类对象可以获取到正确构造器
      Child.prototype.constructor = Child

      // 要在上面子类原型重写之后，再给子类原型添加方法
      Child.prototype.getEmail = function () {
        console.log('email is ' + this.email)
      }

      var c1 = new Child('张三', 13, 'zs@qq.com')
      var c2 = new Child('威威', 19, 'ww@qq.com')

      c1.aa = 200
      console.log(c1.aa) // 200
      console.log(c2.aa) // 100

      console.log(c1)
      console.log(c2)

      c2.hobby.push('帅')
      console.log(c1.hobby)
      console.log(c2.hobby)

      console.log(Child.prototype)
      console.log(c1.constructor === Child) // true 获取对象的构造器 Child

后面es6也有继承，是通过class和extend关键字实现的，底层实现也是基于寄生继承实现的

封装继承函数

      function Parent(name, age) {
        this.name = name
        this.age = age
        this.hobby = ['睡觉']
      }

      Parent.prototype.getName = function () {
        console.log('my name is ' + this.name)
      }

      // 子类构造函数
      function Child(name, age, email) {
        // 伪造继承核心（目的：把父类构造函数中的属性设置到子类对象自身空间中）：
        Parent.call(this, name, age)
        this.email = email
      }

      /**
       * 实现子类继承父类
       * @param {function}： 子类构造函数器
       * @param {function}： 父类构造函数器
       * @return void
       *
       */
      function inherit(sonCtor, parCtor) {
        // 寄生继承路径： 空函数原型  -> 父类原型 -> 子类原型被空函数实例重写
        function F() {}
        F.prototype = parCtor.prototype
        sonCtor.prototype = new F()
        //  修正子类原型对象中的constructor属性，保证原型链的完整性，让子类对象可以获取到正确构造器
        sonCtor.prototype.constructor = sonCtor
      }

      inherit(Child, Parent)

      var c1 = new Child('俊哥', 50, 'junge@qq.com')
      var c2 = new Child('俊哥', 50, 'junge@qq.com')
      c1.getName()
      c1.hobby.push('敲代码')
      console.log(c1.hobby)
      console.log(c2.hobby)

19-玩转JQuery

玩转jQuery

什么是jQuery?

jQuery是一个快速、简洁的JavaScript库(工具箱)。

简单的说就是封装了JS中常用的功能代码，更便于我们的DOM操作、事件绑定。

如：获取页面上面的一个id为box的元素：

  • js原生: var box = document.getElementById('box');
  • jQuery方式: var box = $("#box");

jQuery设计的宗旨是"write Less，Do More"，即倡导写更少的代码，做更多的事情。

使用jquery不用去考虑IE的兼容性问题，它已经帮助我们做了大部分的兼容性。

注！jquery-2.0以上版本不再支持IE 6/7/8) 并不是最新的版本就最好的，而是根据您项目需求所适合的版本！

引入jQuery后，在控制台中输入$.fn.jquery可以查看当前使用的版本号，或者直接看源码

jQuery选择器

什么是选择器？

就是选择页面上元素的一种工具。

  • 类似于js中获取指定id元素，document.getElementById();
  • 类似于js中获取指定class元素，document.getElementsByClassName();
  • 类似于js中获取指定标签元素，document.getElementsByTagName();

上面是js元素的选择器(类似于css选择器)，换成jQuery方式如下：

  • $(css选择器)
  • $("#id")
  • $(".class")
  • $("div")
  • $("ul li ")
  • $("ul > li")
  • $("div[sid=2]")
  • $("[sid=2]")

  • 层级选择器 $("parent > child") 或 $("parent childs")
  • 标签加属性组合选择器 $("ele[attr=value]")

DOM对象和jquery对象

  1. 什么是DOM对象

就是通过原生的js中的方法来获取到的对象，获取到的对象都称之为DOM对象。如下：

  • document.getElementById()
  • document.getElementsByTagName()
  • document.getElementsByName()
  • document.getElementsByClassName()

2. 什么是jQuery对象

若通过jQuery选择器$("选择器")获取到的对象都称之为jQuery对象(本质就是伪数组)。

3. DOM对象和jQuery对象的关系

  • jQuery对象本质是一个伪数组，而DOM对象是以数组下标的形式存在jQuery对象中

4. DOM对象和jQuery对象互相转换

问题？

  • jQuery对象与DOM对象的方法是否能够相互的使用？
  • jQuery对象是否能够使用DOM对象的方法、属性、事件？
  • DOM对象能否使用jQuery对象的方法？

答:都是可以的。只要DOM对象和jQuery对象之间进行转化，即可调用对方的Api。

jQuery对象转化为DOM对象：

  • 方式一:（推荐）
 jQuery对象[下标值]=>获取到对应下标的DOM对象
  • 方式二:
 jQuery对象.get(下标值)=>获取到对应下标的DOM对象

DOM对象转化为jQuery对象

$(DOM对象)=>将DOM对象转为jQuery对象

问题：事件处理程序中的this是DOM对象还是jquery对象？

答：this是当前元素的DOM对象，$(this)才是jQuery对象。

总结：

  1. jquery对象[下标] =》 可以转成dom对象
  2. $(DOM对象) =》 将dom对象转为jquery对象
  3. 事件处理程序中this是指向当前标签的DOM对象，需要$(this)操作才可以转成jQ对象
  4. jq对象获取值时，仅会获取匹配集合中第一个元素的值。但是设置都是批量设置

事件

页面加载事件

  • 原生js语法:
window.onload=funtion(){
    //do to ...
}
  • jQ语法:

$(function(){   
    //do to ...
});

//或

$(document).ready(function(){
    //do to ...    
});

onload和$(function(){ })的区别：

  • onload：等待页面元素（含外部资源img、js）加载完毕后执行
  • $(function(){ })：等待页面元素（仅结构）加载完毕后执行 ,相当于 原生DOMContentLoaded事件

但是在JQ高版本3.6.1中和onload效果是一样的。

面试题： 原生onload和DOMContentLoaded事件的区别

DOMContentLoaded 要快于 onload，因为不需要等待外部的资源

事件绑定

注：在jQuery中事件都是不要加on前缀，原生js需要加on前缀

1、通过事件名直接绑定。如：

jQuery对象.事件名( function(){} )

2、使用bind方法进行事件的绑定，unbind和off进行事件解绑

jQuery对象.bind(type,callback) 
jQuery对象.off(type)/unbind(type)  解绑指定类型的事件
jQuery对象.off()/unbind() 解绑所有事件

参数说明：

  • type：事件名 。
  • callback：事件的处理回调函数

3、使用on方法进行事件的绑定：

jQuery对象.on(type,callback)

参数说明：

  • type：事件名 。
  • callback：事件的处理回调函数
  <ul>

事件委托[on]

注意：上面三种方式都只能给页面已存在的元素绑定事件。对于通过js动态新增的元素是无效的。

解决办法：on方式可以实现事件委托（事件代理）

事件委托的调用形式：

父选择器.on(type,子选择器,callback);

参数说明：

  • type ：事件名
  • selector: 子选择器
  • callback：事件的处理回调函数

对于后续动态新增的子选择器元素同样也是有事件处理效果的。

DOM常用操作

文本操作

基本用法:

  • html() / text() : 操作双标签元素的值，即非表单元素。
  • val() ： 操作表单元素的值。表单元素有（input、select、radio、checkbox、textarea等）：

上面方法特点： 传值就是设置(批量设置)，不传就是获取（第一个对象）

css样式操作

基本用法:

  • css("color") : 获取color的值。
  • css("color"',"red") : 设置color样式为red值。
  • css( {key1:value1,key2:value2} ) : 设置多个样式,传入的参数是一个json对象。

案例：隔行换色

class类名操作

一个元素是可以拥有多个类名的。

<div class="box1 box2 box3">八戒</div>

基本用法:

  • addClass("className ") ：添加一个或多个类名，添加多个用空格隔开
  • removeClass("className") ：删除类名为className
  • hasClass('className')：检查元素是否含有类名为className，有则返回true,否则false.
  • toggleClass("className") ：切换class类名为className,无则添加,有则删除

属性的操作

  • attr(key): 获取key属性,key是属性名称，可以是自定义属性也可以是内置属性
  • attr(key,value): 设置属性key,值为value
  • removeAttr(key) : 删除对象的key的属性
  • prop(key,value); 只能设置内置属性如title、id、disabled,checked

特别注意：

jQuery选择器匹配到的对象如果是多个，设置他们内容、样式等都是批量设置。

但是获取的时候只会获取到默认第一个对象的值。

jQuery中的链式编程

链式编程：允许在一条语句中运行多个jQuery方法。

形如：对象.方法().方法().方法().......

如：$("#box").css('color','red').html('刘德华')

注：是否可以继续向后链式调用，得看前面的方法是否返回一个对象(return this)，是的话就可以继续调用其对象的相关方法或属性。

以下是常用的用于链式编程的节点查找方法：

  • $("li:eq(0)") 或 $(li:first) 或 $('li').eq()完全一样， 获取下标为0的jquery对象
  • last()：获取父的最后一个元素
  • first()：获取父的第一个元素
  • eq(index) ：获取指定下标的元素
  • parent()：获取当前元素的父元素
  • parents([selector])：获取当前元素满足条件的祖先元素
  • siblings([selector]）：获取当前元素所有满足选择器条件的兄弟元素，不包含自己。
  • children([selector])：获取满足所有选择器条件的子元素，没有指定选择器，则获取所有的子元素
  • find(selector):获取当前元素下满足条件的子孙元素

    其中参数selector写选择器可以进行筛选。

链式调用注意的地方

  • 给对象设置一些值如样式或类名依然可以继续往后链式调用，因为这类方法还是返回当前的jQuery对象。 如： 
addClass('className') 设置类名
css('color','red')    设置样式
html('content')    设置内容
attr('sid') 设置内容
......
  • 调用某些方法如果不是返回jQuery对象，如返回一个字符串。则不可以继续向后链式调用。 如： 
 - html()：获取标签的值
 - css('font-size'); 获取字体大小

重点：所以判断一个方法是否可以继续向后链式调用得看前一个方法是否返回的是jQuery对象。

jQuery动画

显示和隐藏

  • show()： 显示元素
  • hide(): 隐藏元素
  • toggle(500,function(){}) 在显示和隐藏间进行切换

滑动效果

  • slideDown()：将隐藏的元素显示（滑）出来
  • slideDown(speed,[callback]) ：以设定speed的速度将隐藏的元素显示出来
  • slideUp()：将显示的元素隐藏
  • slideUp(speed,[callback]) ：以设定speed的速度将显示的元素隐藏
  • slideToggle()：在显示或者隐藏间切换
  • slideToggle(speed,[callback]) ：以设定speed的速度显示或者隐藏

其中speed是动画的速度，这个参数有两种表示方式 ：毫秒数或英文单词如"slow", "normal", "fast"

淡入淡出

  • fadeTo(speed,opacity)：设定指定的时间实现淡入或淡出到指定的透明度，

opacity:透明度，取值为0至1之间。

  • fadeIn()：淡入（显示）
  • fadeOut()：淡出（隐藏）
  • fadeToggle()：在淡入或者淡出之间进行切换

自定义动画

  • animate(styles,[speed],[callback])：在规定的时间内 完成动画的执行！

参数说明：

  • styles：要求是一个对象,样式的值和css方法设置是一样的，其中样式值只有数字值才可创建动画（如：font-size:30px，left:30）,字符串值无法创建动画（如：background-color:red）。
  • speed：动画完成所需要的时间
  • callback：完成动画后执行的回调函数

在以后真正的实际项目，如果要用到动画类效果，一般情况都是一些UI库已经集成好的，或者也可以借助一些第三方有名动画库如：Animate.css

jQuery中插件机制(插件开发)

利用插件机制可以给jquery对象扩充更多的方法

语法：

// 扩充多个方法
$.fn.extend({
     方法名1:function(){},
    方法名2:function(){}
});

// 若是单个方法可以这么写
$.fn.方法名 = function(){}
jQuery.fn.方法名 = function(){}

// 调用：
jquery对象.方法名();

注意：在jQuery插件机制中的方法其this指向有点特殊，它指向的是当前jQuery对象 。但在其它方法中的this都是指向当前DOM对象，如事件绑定中

示例：

// 扩充jquery对象方法
$.fn.foo = function(){
    console.log('foo被被调用了')
} 

$("#btn").foo();

// 扩充jquery本身,本质就是一些静态函数
$.bar = function(){
    console.log('bar called')
}
$.bar();

如：给jquery本身扩充一个方法，判断是否是原始类型

// 给jquery本身扩充一个方法，判断是否是原始类型
$.isPrimitiveType = function (data) {
// number、string、boolean、symbol
    return (
        typeof data === 'number' ||
        typeof data === 'string' ||
        typeof data === 'boolean' ||
        typeof data === 'symbol'
    )
}

console.log(  $.isPrimitiveType({}) ); // false
console.log(  $.isPrimitiveType(1) ); // true

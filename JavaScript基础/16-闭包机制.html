16-闭包机制

什么是闭包(closure)【非常重要】

闭包：闭包是指有权访问另一个函数作用域中的变量的函数。

先看下面的一段代码：

function A(){
  var a = 10;
  function B(){
    console.log(a);
  }
  return B;
}
var result = A();
result();  // 10

解释：

由于只有内部函数B才可以访问到函数A中的局部变量，那么我们只需要把函数B作为函数A的返回值，就可以在函数A外部访问它的内部变量！ 其中函数B就是闭包。

闭包如何产生的？

函数内部只要访问了父函数的标识符（变量名、函数名）就会产生闭包

专业点解释：函数内部读取变量首先在自身作用域环境中查找它的声明，如果没有找到会向父级作用域中查找。一旦读取了父级作用域中的变量就会产生一个闭包。

如何创建闭包？

创建一个父函数，父函数内在创建子函数，子函数读取父函数内的局部变量，最后返回子函数

这是创建闭包最常见的方式。

闭包的作用

保存变量的作用。

即保持对函数内部的变量持续引用，让一个变量始终保存在内存中，即闭包可以延长变量的生命周期。

怎么理解“保持对函数内部的变量持续引用，让一个变量始终保存在内存中”这句话呢， 请看以下代码：

function A(){
  var a = 10;
  function B(){
    console.log(a++);
  }
  return B;
}
var result = A();
result(); // 10
result(); // 11
result(); // 12

解决for中使用定时器问题：

for(var i = 1; i <= 3; i++ ){
  (function(i){
    setTimeout(function(){
      console.log(i);
    },1000)
  })(i)
}

        for (var i = 0; i < lis.length; i++) {
            // IIFE：保存变量i
            (function (i) {

    </script>

闭包的缺点

由于闭包会使得函数中的变量都保存在内存中，内存消耗很大，所以不能滥用闭包，使用不当可能会导致内存泄漏问题。

内存泄漏：即内存无法释放。由于疏忽或错误造成程序未能释放已经不在使用的内存引起的。 内存溢出：内存被撑爆。

function fn(){
  var a = 1;
  function fn1(){
    a++;
    console.log(a)
  }
  return fn1;
}

var f = fn();
f(); // 2
f(); // 3
f = null; // 不再使用，手动释放内存

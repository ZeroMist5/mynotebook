11- try...catch异常处理

认识错误（error）

错误难免会发生！

当执行 JavaScript 代码时，会发生各种错误。 错误可能是程序员的编码错误，或者不严谨，或者由于其他不可预见的问题导致的。 但是这些错误我们是不想展现给用户看的，所以我们开发者需要对其进行处理,进行解决。

如：打印一个不存在的变量abc，会抛出ReferenceError

console.log(abc)
Uncaught ReferenceError: abc is not defined // 未捕获的引用错误：abc未定义

如：打印一个非法的字符串：

console.log('fdfdf);
Uncaught SyntaxError: Invalid or unexpected token; // 未捕获的语法错误：无效或意外标记

如：类型错误：

undefined.toString();

// undefined不能调用toString
Uncaught TypeError: Cannot read properties of undefined (reading 'toString')

上面的错误都是由于程序员自身编码导致的，这些js错误会影响程序正常的执行。所以我们要对这些意外的错误进行捕获处理，从而让程序正常执行。

try-catch-finally机制捕获错误

JavaScript语言提供一种捕获错误的机制try-catch-finally，很多语言都有这种机制。如java、php等。

代码演示， （1）try-catch-finally介绍

  • try：用来捕获代码块中可能会发生的错误。
  • catch： try中产生的错误，会进入到catch中，这里可以对错误进行相应处理
  • finally： 在 try 和 catch 之后执行，无论try是否错误都一定会执行。

完整结构代码：

try{
     // 用来捕获代码块中可能会发生的错误
}catch(err){
    //只有try{}中代码有异常，才会执行这里的代码,这里可以对错误进行相应处理
}finally{
    //无论try中的代码是否异常，这里的代码一定会执行，
}

一般finally不用，try-catch组合使用较多

示例代码：

try{
    console.log(1)
    console.lo(abc)； // 出错，下面代码不会执行，直接进入到catch
    console.log(2)
}catch(err){
    console.log(3)
    console.log('错误对象',err)
}finally{
    console.log(5)
}

在js中，不同的错误抛出的错误对象类型也是不同的。catch中的err参数就是当前的错误对象

常见的错误对象：

错误对象 说明
ReferenceError 访问不存在的变量或方法时
SyntaxError 语法错误
RangeError 数值变量或参数超出其有效范围
TypeError 变量或参数不属于有效类型
EvalError 与eval()函数相关操作的错误
URIError 给 encodeURI() 或 decodeURI() 传递的参数无效
...

详细参见：Error

错误对象的常用属性

  • Err.name: 获取错误类型
  • Err.messsage: 错误的详细信息
  • Err.stack: 获取错误堆栈信息。用于获取错误发生源头，找bug有利。
try {
  var num = 500;
  console.log(abc);
} catch (err) {
  console.log('错误信息:', err.message);
} finally {
  console.log('finally');
}

错误的正确处理

有错误，应该进行错误收集和上报。

  • 收集: 把错误信息写入到错误日志文件中，供以后排查bug
  • 上报： 把错误上报给服务端，服务端可以对此错误进行发生邮件或短信给开发者，开发者及时进行修复
  • 其他相应逻辑
catch (err) {
  // 收集错误和上报错误
}

遇到错误心态：

  1. 不要慌
  2. 看错误信息（查英文）
  3. 避免下次再犯

遇到的bug越多，后面解决问题的能力也会越强！

throw-抛出自定义错误

throw语句允许我们抛出一个自定义错误。

基本语法格式：

throw new Error('错误信息')

问题：什么时候主动抛出一个错误给开发者？ 答：由于某段代码可能会发生错误（悲观的认为），这时候我们就可以抛出一个错误提示开发者。

function sum(n1,n2){
    if(typeof n1 !== 'number' || typeof n2 !== 'number'){
        throw new Error('参数必须是number类型')
        console.log('这里不会执行'); // 上面抛出一个错误后，其后面代码是不会执行的
    }
    return n1 + n2;
 }

console.log(  sum(10,20) ); // 30
console.log(  sum('20',20) ); // 抛出错误

外部引入的js脚本发生错误：

注意：当一个html文件引入一个外部的js文件时，如果此js文件抛出一个错误，则此js文件内后续的代码不会执行，但是不会影响html文件内script段中代码的执行。

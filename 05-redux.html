05. redux

redux

redux介绍

​ redux是一个用于集中管理状态数据的工具（react中多个组件用到的公共状态就可以让redux管理）

注意：redux是独立于react的，本质就是一个独立的js库

​ redux使用的时候，有三个重要的角色,可以这么理解： store(项目经理)：派发需求给reducer reducer(程序员)：完成store分配的需求 action(产品经理)：提出意见给store

redux的基本使用

1、下载包 npm i redux 2、创建store对象：

import {createStore} from 'redux' const store = createStore(reducer 函数)

3、创建reducer函数，并传入到创建store的代码中

实现代码如下：

// index.js
const {createStore} = require('redux')

/* 
reducer的执行时机：
    1、当createStore时，reducer函数会被调用一次，目的就是为了初始化store的数据
    2、调用了store.dispatch（action对象）
*/

// 定义reducer函数
// state中作为形参，写了一个默认值的状态
function reducer(state={count:0,msg:'计数'},action) {
    // state就是redux中的源数据
    // 假设state是{count:0,msg:'计数'}
    // action对象，用来描述需求,有一个要求，至少需要有一个type属性
    switch(action.type) {
    case 'add':
        // 注意：reducer函数返回什么则store中的state就会被改成什么，是覆盖的操作
        return {
            ...state,
            /*  ...state是展开state对象，
                相当于是：
                count:0,
                msg:'计数',
                count:state.count + 1
                由于对象中如果会有同名属性的存在，那么后面的属性会覆盖前面的属性
            */
            count:state.count + 1
        }
    case 'sub':
        const newCount = state.count - 1
        return {
        ...state,
        count:newCount
        }
    // 不管有没有case，或有多少个case，都必须要写一个default
    // 写这个default是为了再创建store对象时初始化数据
    default:
        return state
    }
}
const store = createStore(reducer)

// 调用了store.dispatch（action对象），reducer函数就会执行 store.dispatch()

使用高阶组件的方式来优化计数器案例

​ 由于考虑到实际项目开发的时候会有多个组件需要用到redux的数据，所以下面的代码会需要在每个组件当中都写一遍，这显然是不合理的，所以可以使用高阶组件来优化

    const [a,setA] = useState(0)
     useEffect(() => {
         store.subscribe(() => {
         console.log('store的数据改变了',store.getState())
         setA((a) => a + 1)
    }

定义一个公共的组件，再把公共部分的代码写入进去，然后在引入到App组件中，再new

// Far.js
import React,{useState,useEffect} from 'react'
export default function (Comp) {
    return function Far() {
        const [a,setA] = useState(0)

        useEffect(() => {
            store.subscribe(() => {
                setA((a) => a + 1)
            })
        },[])
        return <Comp {...props}></Comp>
    }

}

// App.js
import React from 'react'
import Count from './Count'
import Test from './Test'
import withFar from './Far'

const WithCount = withFar(Count)
const WithTest = withFar(Test)
export default function App() {
    return (
    <div>
        App
        <hr />
        <WithCount></WithCount>
        <WithTest></WithTest>
    </div>
    )
}

使用自定义Hook来优化

​ 如果是用高阶组件的话，是能够解决。但是代码中是用的函数组件，所以没有必要再使用类组件方面的，所以可以使用自定义hook来优化

import React,{useState,useEffect} from 'react'
import store from './redux/store'
export default function useRedux() {
    const [a,setA] = useState(0)

    useEffect(() => {
        store.subscribe(() => {
            setA((a) => a + 1)
        })
    },[])
}

需要用到上述代码，只需要引入此文件，然后调用useRedux()即可

react-redux的使用

react-redux使用方式一

​ 由于redux是一个独立js库，如果react中使用redux，则当redux数据发生变化的时候，react组件并不会自动更新，从而渲染最新的数据，所以需要使用react-redux这个包来帮我们将react和redux连接起来，当redux数据变化了，则react组件会自动更新。（内部的实现原理就是使用的react高阶组件）

使用步骤 1、下载包 npm i react-redux 2、在入口文件中，引入一个组件叫做Provider，使用Provider包裹App，并且给Provider添加store属性，值为redux中的store对象

3、哪个组件需要用到redux的数据，则哪个组件中从react-redux中引入一个函数叫做connect

// index.js
import React from 'react'
import ReactDOM from 'react-dom/client'
import {Provider} from 'react-redux'

import App from './App'
import store from './redux/store'

ReactDOM.createRoot(document.getElementById('root')).render(
    // 作用：是把store传递给项目中所有的和redux关联的那个高阶组件
 <Provider store={store}>
       <App />
    </Provider>
)

使用方法： import {connect} from 'react-redux' const WithCount = connect()(Count)

react-redux使用方式二

​ 使用方式和方式一的前两个步骤是一样的(下包和引用Provider包裹App组件)，只是不用connect，取而代之的是两个hook，因为函数组件的出现，所以渐渐的成为了旧技术，用的很少

分别是useSelector 和 useDispatch useSelector的作用：帮我们监听redux数据的变化，并且过滤数据 useDispatch的作用：返回store.dispatch函数

// 使用方式
// 引入useSelector、useDispatch
import { useSelector, useDispatch } from 'react-redux'
export default function Count() {
    const count = useSelector((state) => {
        // 返回值就是回调中返回的数据
        return state.count
    })
    const dispatch = useDispatch()
    return (
        <div>
            {count}
            <hr />
            <p></p>
            <button
              onClick={() => {
                dispatch({type:'add'})
    }}
    >
        +
    )
}

redux实际使用场景

​ redux文件夹下，除了reducer.js、store.js还有两个文件分别是actions.js和constants.js

对于实际开发中，例如计数器中的值并不是固定的甚至已知，所以我们给值定死是不合理的。所以后面的值一般是一个变量，规范之中是定义成变量payload

​ actions.js里面用于封装一些函数，返回action对象，因为action对象的数据会越来越长，每次dispatch调用都需要写一长串的action对象，写起来太繁琐，后面就通过封装一个函数，返回一个action对象，再把这些函数都放在action.js文件中，后面只需要调用这个函数就能够通过该函数的返回值得到这个action对象，代码就变得很简洁

​ 这些函数有一个统一的名字，叫做actionCreator

constants.js，在开发的时候action对象中的type值字符串可能会很长，类似于'add'、'sub'等这些，在代码编辑器中是没有提示的，特别容易写错。所以，定义一个constants.js文件，专门用于存放这些变量，变量命名的时候建议是大写

redux-thunk

​ 场景：当使用React组件中，如果需要完成异步操作的功能，代码上是允许把异步代码写在表单标签的onClick中，实际开发中规范是不允许这样来操作redux数据。那么该怎么实现既不写在React组件，又能使用异步操作redux数据呢？

reducer函数中不能执行异步，也被称为纯函数

什么是纯函数？输入同样的参数，得到相同的结果 例如： function fn(x) { return x + 1 } 每次传入1的时候，函数的返回值恒等于2

结论：redux默认情况下，不能支持异步

但是结合redux-thunk就可以实现异步

使用步骤： 1、下载包，npm i redux-thunk 2、在store.js引入redux-thunk，并进行配置 3、在dispatch中传入一个函数，这个函数中才是那个真正的中间件。如果没有配置applyMiddleware(thunk)，直接在dispatch中传入一个函数则会报错，如下图： 

import {createStore,applyMiddleware} from 'redux'
import reducer from './reducer'
import thunk from 'redux-thunk'

const store = createStore(reducer,applyMiddleware(thunk))
store.subscribe(() => {
console.log('store的数据改变了',store.getState())
})
export default store

dispatch((dispatch) => {
  // 在这里发送异步请求
  setTimeout(() => {
    dispatch() // 这里的dispatch相当于node中中间件的next(),如果是直接用真正的dispatch则会直接调用reducer。简单来说就是，如果redux-thunk后面还会有中间件，那么这个接受的形参dispatch就不是真正的dispatch，反之则是真正的dispatch
  },2000)
})

// actions.js
// 可以把上述代码藏起来，不像是操作的代码放在React组件中
export function addAsync(payload) {
  setTimeout(() => {
      // 一般来说dispatch调用是立即执行reducer函数，但是由于有中间件，那么就先执行传入的回调
    dispatch(add(payload))
  },2000)
}

actions.js文件中，返回一个action对象的函数称为同步action 返回一个函数的称为 异步action

合并reducer

​ 实际开发的时候，为了避免代码冲突，每一个程序员都会写自己的reducer函数，但是创建stroe时，只能传入一个reducer函数，所以就需要将多个reducer函数合并成一个

使用redux中的一个包 —— combineReducers,返回的是一个reducer const rootReducer = combineReducers({ zs:countReducer, ls:listReducer })

注意：在合并reducer之后，数据的结构会发生变化，所以在使用到此数据的时候，调用useSelector的时候返回的对象需要进行响应的更改

@reduxjs/toolkit(redux新版本的写法)

@reduxjs/toolkit是一个基于之前redux包二次开发的包

使用@reduxjs/toolkit原理和之前学习的redux的原理完全一直，但是语法完全不同

使用步骤 1、下载：npm i @reduxjs/toolkit react-redux 2、在index.js中使用Provider 3、创建store对象

// 创建store
import {configureStore} from '@reduxjs/toolkit'

const store = configureStore({
    reducer:{
        count:countSlice.reducer, // 就是自动生成reducer函数
    }
})

// redux新版本中的slice，其实就是reducer和actions的集合体
import {createSlice} from '@reduxjs/toolkit'

const countSlice = createSlice({
    name:'xxx',
    initalState:{count:0,msg:'hello'},
    reducers:{
        add(state,action) {
            // add函数中拿到的state数据不是真是的store数据，而是store数据的代理，所以可以直接修改
            state.count = state.count + action.payload
        }
    }
})
export default countSlice

// index.js 入口文件
import React from 'react'
import ReactDOM from 'react-dom/client'
import {Provider} from 'react-redux'
import store from './redux/store'

ReactDOM.createRoot(document.getElementById('root')).render(
    <Provider store={store}>
        <App />
    </Provider>
)

// Count.js
import React,{useState,useEffect} from 'react'
import {useSelector,useDispatch} from 'react-redux'
import countSlice from './redux/slice'
const {add} = countSlice.actions
export default function Count() {
    const count = useSelector((state) => {
        return state.count.count
    })   // 这是slice里面自动生成的actionCreator
    const dispatch = useDispatch()
    return (
        <div>
          count
          <hr />
          <p>{count}</p>
          <button
            onClick={() => {
               dispatch(add(1))
            }}
          >
           +
          </button>
        </div>
    )
}

使用新版本来实现异步操作

​ 如果需要实现异步操作，那么只要新增代码

// 创建异步对象，这个对象有两个作用：
    1、相当于就是一个异步action
    2、asyncCount可以监听返回的promise的状态
const asyncCount = CreateAsyncThunk('yyy',() => {
    return new Promise((resolve,reject) => {
        // 这里执行异步操作
        setTimeout(() => {
            resolve(5)
        },2000)
    })
})
// 配置异步操作
extraReducers(builder) {
    builder.addCase(asyncCount.pending,(state,action) => {
        console.log("异步开始执行了")
    })
    builder.addCase(asyncCount.fulfilled,(state,action) => {
        console.log("异步执行成功了")
    })
    builder.addCase(asyncCount.pending,(state,action) => {
        console.log("异步执行失败了")
    })
}

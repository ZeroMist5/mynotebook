flex弹性盒子

弹性盒子布局

也叫弹性布局、弹性盒子、flex布局、伸缩布局、伸缩盒子。

弹性盒子布局是一种简单、强大的布局方式（可以指明空间的分布方式、内容的对齐方式、元素的视觉顺序（在不更改源码顺序的基础上））。

弹性布局是父元素和子元素之间的关系。

父元素：弹性空间或弹性容器。

子元素：弹性项或弹性元素。

弹性容器

弹性容器属性比较多，经常用到的属性都是给弹性容器上面设置的。

父元素设置display:flex

设置了display:flex之后默认的时候可以让其中的子元素横向排列。默认的时候当父元素宽度小于子元素的宽度总和的时候子元素将会自动伸缩。

<!DOCTYPE html>
<html>

<head>
    <style>
        #f{
            display:flex;
            border:2px solid green;
        }
        #f div{
            width:100px;
            height:100px;
        }
        #f div:first-of-type{
            background-color:lightcoral;
        }
        #f div:nth-of-type(2){
            background-color:greenyellow;
        }
        #f div:nth-of-type(3){
            background-color:lightseagreen;
        }
        #f div:nth-of-type(4){
            background-color:skyblue;
        }
        #f div:nth-of-type(5){
            background-color:red;
        }
        #f div:nth-of-type(6){
            background-color:pink;
        }
    </style>
</head>

<body>
    <div id="f">
        <div>1</div>
        <div>2</div>
        <div>3</div>
        <div>4</div>
        <div>5</div>
        <div>6</div>
    </div>
</body>

</html>

弹性元素

弹性容器的子元素称为弹性元素（只有直接子元素受弹性盒子布局的影响），弹性容器的子元素成为弹性元素之后也可以变成弹性容器。

在弹性容器中，各个弹性元素主轴（主轴可以是横向的也可以纵向的）四面排列。

  • 主轴：内容沿着此轴开始流动，他指明了弹性元素的流动方向。默认的主轴是水平方向，左边开始右边结束。

  • 垂直轴（交叉轴、侧轴、辅轴）：弹性元素沿着侧轴堆叠，指明了新弹性元素换行的方向。

    垂直轴始终与主轴垂直

    默认的时候垂直轴从上面开始、下面结束。

设置主轴方向

flex-direction，用来指定在弹性容器中如何摆放弹性元素

弹性元素沿着主轴的起始边开始设置，流向主轴的结束边。

  • row，主轴的起始边在弹性容器左边，结束边在弹性容器右边。默认值。

  • row-reverse，主轴的起始边在弹性容器右边，结束边在弹性容器左边（像右浮动）。

  • column，主轴的起始边在弹性容器上边，结束边在弹性容器的下边。垂直轴变为横向的，左边开始右边结束。

  • column-reverse，主轴的起始边在弹性容器下边，结束边在弹性容器上边。垂直轴变为横向左边开始右边结束。

    如果一行有空白，那么空白出现在主轴结束的方向。

父元素和子元素的关系。

前面说的两个属性（都需要加在父元素上）：

  • display:flex开启弹性盒子。

  • flex-direction确定主轴方向。

    • row，起始边在弹性容器左侧，结束边在弹性容器右侧。
    • row-reverse，起始边在弹性容器右侧，结束边在弹性容器左侧。
    • column，起始边在上边，结束边在下边。垂直轴变成横向的左边开始右边结束。

    • column-reverse，起始边在弹性容器下边，结束边在弹性容器上边。

      弹性元素沿着主轴方向流动，一行里面如果有空白，放在主轴的结束方向。

主轴：确定每一行的弹性元素应该怎么摆放。

垂直轴：控制换行的方向。

width:100%和width:auto：

  • width如果是100%那么，在没有box-sizing:border-box的情况下，padding、border都会在100%的基础上增加，就会超出父元素。如果说有box-sizing:border-box。那么设置width:100%等于内容区宽度+padding+border
  • 如果width:auto的话会自动进行容纳块公式的计算。

换行

弹性元素在弹性容器的主轴放不下，默认情况下弹性元素不会换行。

flex-wrap来控制换行（用在弹性容器）。

  • nowrap，沿着主轴在一行上面排列，不会换行。

  • wrap，如果超过弹性容器，会沿着垂直轴的方向进行换行。

  • wrap-reverse，如果超过弹性容器将会沿着垂直轴的反方向进行换行。

    设置弹性元素如何在主轴上进行分布

    默认情况下弹性容器中放完全部弹性元素留下的空白出现在主轴结束边（从主轴的起始边开始放置的）。

    可以使用justify-content属性来设置在弹性容器主轴上如何分布各行里面的弹性元素。

    • flex-start，默认值。弹性元素紧靠主轴的起始边。

    • flex-end，弹性元素紧靠主轴的结束边。

    • center，将弹性元素作为一个整体，居中显示在主轴的中点。

    • space-between，将每一行里的第一个弹性元素放在主轴起始边，最后一个弹性元素放在主轴结束边，余下的每一对相邻的弹性元素之间放置等量的空白。

      余下的空间/(弹性元素个数-1)

    • space-around，计算弹性元素的数量，将剩余的空间分配到每个弹性元素上，得到这些空间之后再除以2，分别放在弹性元素的左边和右边。

      (余下空间/弹性元素个数)/2

    • space-evenly，计算弹性元素的数量，在这个基础上+1，然后把每一行里面的空间分成这么多分。每一个里面的每个弹性元素前面放置一份，最后一份放在一行里的最后一个弹性元素后面。

      余下的空间/(弹性元素的个数+1) 得到的值每个弹性元素前面放一份，最后一份放在最后。

      justify-content即使一行放不下换行了，也会在新开的行上面重新进行计算并且按照对应的公式进行排列。

设置弹性元素如何在垂直轴上分布

align-items，这个属性定义的是弹性元素在垂直方向的对齐方式（设置弹性元素在当前行的位置分布）

  • flex-start，将弹性元素与弹性容器垂直轴起始边对齐。

  • flex-end，将弹性元素与弹性容器垂直轴结束边对齐。

  • center，中间对齐。

  • stretch，默认值。如果弹性元素没有设置高度或高度为auto，将沾满当前行的高度。

    如果想单独的修改某个弹性元素的对齐方式，可以在这个弹性元素上使用align-self属性。

    align-self

    • auto，默认值。algin-self的值使用align-items的。
    • flex-start
    • flex-end
    • center

    • stretch

设置整个弹性容器中的元素如何垂直对齐

align-items是对每行进行对齐。align-content将整个弹性容器中的元素看成一个整体然后进行对齐。

align-content属性对单行的弹性盒子无效，也就是说你必须要换行。但是具体有几行不做限制

<!DOCTYPE html>
<html>

<head>
    <style>
        #f{
            height:500px;
            width:300px;
            border:1px solid green;
            display:flex;
            flex-wrap: wrap;
            align-content:flex-end;

        }
        #f div{
            width:100px;
            height:100px;
        }
        #f div:nth-of-type(1){
            background-color:lightcoral;
        }
        #f div:nth-of-type(2){
            background-color:darkgoldenrod;
        }
        #f div:nth-of-type(3){
            background-color:lightgreen;
        }

    </style>
</head>

<body>
    <div id="f">
        <div>1</div>
        <div>2</div>
        <div>3</div>
    </div>

</body>

</html>
  • flex-start，弹性元素紧靠垂直轴起始边。
  • flex-end，紧靠垂直轴结束边。
  • center，将弹性元素作为一个整体，居中显示在垂直轴的中点。
  • space-around
  • space-evenly
  • space-between，第一行里的将弹性元素放在垂直轴起始边，将最后一行里的弹性元素放在垂直轴结束边。余下的每一行弹性元素之间放置等量空白。
  • stretch。如果不设置高度或高度为auto，那么沾满当前行。

align-content存在时align-items不起作用。

align-content和align-items的区别：

  • align-content将所有元素看成一个整体然后进行操作。
  • align-items是针对于每一行如何进行操作。

float不会浮动弹性元素。

对弹性元素使用position进行定位的时候和前面讲的一样。

  • relative依然相对于自身进行定位。
  • absolute、fixed依然脱离文档流。但是要注意absolute、fixed脱离文档流之后默认位置是可以使用flex属性控制的。
<!DOCTYPE html>
<html>

<head>
    <style>
        #f{
            height:1000px;
            width:300px;
            border:1px solid green;
            display:flex;
            flex-wrap: wrap;


        }
        #f div{
            width:100px;
            height:100px;
        }
        #f div:nth-of-type(1){
           position:absolute;
           align-self:center;
           /* right:0;
           bottom:0; */
            background-color:lightcoral;
        }
        #f div:nth-of-type(2){
            background-color:darkgoldenrod;
        }

    </style>
</head>

<body>
    <div id="f">
        <div>1</div>
        <div>2</div>

    </div>

</body>

</html>

弹性增长因子

上面所说的属性除了align-self都是在父元素上用的。

flex-grow，适用于弹性元素。用来定义有多余的空间是否允许弹性元素增大，以及增大的比例。

默认值是0，可以使用其他的数值（不能用负数），可以是小数。

你家里有三个孩子：家里有750W，每个孩子本来一个人先分100w，剩下的450w应该怎么分给三个孩子。

弹性元素排序

默认的时候，弹性元素的显示顺序和排序顺序在源码中是一致的。

order属性可以用于修改单个弹性元素的显示顺序的。

格式： order:value，用于弹性元素。确定显示顺序的。

可以将order设置为一个非零的整数。默认的情况下，所有弹性元素的顺序都是0。

order的值越大越在主轴上靠后显示。如果数值相同则在一个排序组中，然后以在源码中出现顺序沿着主轴显示。

先比较的分组，分组里面的order值都相同然后比较谁先出现，先出现的放在前面。

<!DOCTYPE html>
<html>

<head>
    <style>
        #f{
            height:500px;

            border:1px solid green;
            display:flex;

        }
        #f div{
            width:100px;
            height:100px;
        }
        #f div:nth-of-type(1){
            order:1;
            background-color:lightcoral;
        }
        #f div:nth-of-type(2){
            order:1;
            background-color:darkgoldenrod;
        }
        #f div:nth-of-type(3){
            background-color:lightgreen;
        }
        #f div:nth-of-type(4){
            background-color:yellow;
        }
        #f div:nth-of-type(5){
            background-color:green;
        }
        #f div:nth-of-type(6){
            background-color:pink;
        }


    </style>
</head>

<body>
    <!-- 345612 -->

    <div id="f">
        <div>1</div>
        <div>2</div>
        <div>3</div>
        <div>4</div>
        <div>5</div>
        <div>6</div>
    </div>

</body>

</html>

只是视觉上变化，源代码中的顺序不变。

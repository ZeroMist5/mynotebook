浮动

文档流：文档流指的是HTMl中元素在计算布局、排版过程中的机制。普通文档流就是从上倒下、从左到右来解释。解释的时候如果行内元素在一行显示，如果是块状元素就自己独占一行。

浮动的出现是为了打破正常文档流而设置的，实现文字环绕。

格式：float:value

  • none，无浮动。
  • left，元素向左浮动（其他的内容包围在浮动元素的的右边）
  • right，元素向右浮动（其他内容包围在浮动元素的左边）

利用float形成图片、文字环绕的原理：

  1. 父元素高度塌陷，让跟随的内容可以和父元素在一个水平线上。

  2. 块状元素和浮动元素会重叠，但是块状元素中的行框盒子和浮动元素不重叠。

float属性的规则

  1. 文档源码中处于前面的元素向左浮动，就是后面的浮动元素在前面浮动元素的右侧。同样前面的浮动元素向右浮动，后面的浮动元素会在前面浮动元素的左侧。

     <!DOCTYPE html>
 <html>

 <head>
     <style>
         div{
            /*  float:left; */
            float:right;
             width:200px;
             height:200px;
             border:1px solid green;
         }
     </style>
 </head>

 <body>
     <div>1</div>
     <div>2</div>
     <div>3</div>
 </body>

 </html>

清除浮动

格式：clear:value

沛华老师叫我去喝酒（怕老婆）。

沛华老师叫我去喝酒，我没去。抵抗。。。。

clear属性用来设置自身怎么样，而不是设置的float元素怎么样。

元素盒子的边不能和前面的浮动元素相邻。

  • none，允许任何一边出现浮动。
  • left，抗左侧浮动。
  • right，抗右侧的浮动。
  • both，抗两侧的浮动。
<!DOCTYPE html>
<html>

<head>
    <style>
        div {
            float: left;
            width: 200px;
            height: 200px;
            border: 1px solid red;
        }
    </style>
</head>

<body>
    <div>沛华</div>
    <div style="clear:both;">我</div>
</body>

</html>

clear方法来撑高父元素的高度：

clear应用到非浮动的块状元素上时，它将移动到相关浮动元素的下方。

方法一：直接使用空div标签方式解决高度他先。

            border: 1px solid red;
        }

<body>
    <div id="f">

<!DOCTYPE html>
<html>

<head>

项目布局：css（放置css文件）、js（放置JS文件）、imgs（放置图片），也就是说将这些东西单独放在单个的目录中。

  1. 默认样式问题

    很多的标签都会有默认的样式。但是我们不需要，所以就需要清除默认样式。不同的浏览器有可能呈现的默认样式也不同。

    reset.css或normalize.css用来进行初始化。

    1. 保护了有价值的默认值。
    2. normlaize.css修复了一些浏览器的bug。

  2. 版心和通栏盒子

    页面的版心：PC端的网页通常是一个固定宽度的水平居中的盒子，版心用来显示网页中的主要内容。

    页面版心是按照设计稿来的，一般都在1100px以上1100~1300，如果配上低分辨率的显示器就底部出现滚动条（比较少见），如果是高分辨率的显示器那就两侧留白。

    还有一种盒子叫通栏盒子，通栏盒子是从屏幕最左边到屏幕最右边的盒子。通常用来做装饰。
